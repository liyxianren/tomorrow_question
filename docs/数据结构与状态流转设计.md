# 数据结构与状态流转设计

## 1. 设计目标

本文件定义首版正式规则下的状态模型与状态流。  
核心要求只有两个：

1. 所有关键结果都能进入快照并支持刷新恢复
2. 状态结构直接表达“国家收入分账制”

## 2. 快照主结构

建议的对局快照结构如下：

```ts
type GameSnapshot = {
  gameId: string;
  round: number;
  maxRounds: 15;
  phase: "decision" | "market" | "settlement";
  phaseDeadlineAt: string | null;
  nationalStateByPlayer: Record<PlayerId, NationalState>;
  phaseWorkspace: DecisionPhaseWorkspace | MarketPhaseWorkspace | SettlementPhaseWorkspace;
  ranking: RankingEntry[];
};
```

## 3. 玩家国家状态

```ts
type NationalState = {
  countryId: CountryId;
  domesticSalesRevenue: number;
  overseasSalesRevenue: number;
  nationalIncome: number;
  cumulativeNationalIncome: number;
  incomeAllocationRatio: {
    domesticMarket: number;
    factory: number;
    governmentFiscal: number;
  };
  budgetPools: {
    domesticMarket: number;
    factory: number;
    governmentFiscal: number;
  };
  techPoints: number;
  militaryPoints: number;
  factories: FactoryState[];
  inventory: InventoryState[];
  activePolicies: ActivePolicyState[];
  activeStrategies: ActiveStrategyState[];
  overseasMarketAccess: OverseasMarketAccessState;
};
```

### 字段语义

- `domesticSalesRevenue`
  - 本回合国内销售额
- `overseasSalesRevenue`
  - 本回合海外销售额
- `nationalIncome`
  - 本回合国家收入
- `cumulativeNationalIncome`
  - 全局累计国家收入
- `incomeAllocationRatio`
  - 当前用于财政结算的收入分配比例
- `budgetPools`
  - 三个独立预算池的当前余额
- `techPoints` / `militaryPoints`
  - 当前可支配点数

## 4. 工厂与库存状态

```ts
type FactoryState = {
  factoryId: string;
  routeId: string;
  level: number;
  capacity: number;
  workerCost: number;
  status: "idle" | "scheduled" | "producing";
};

type InventoryState = {
  goodsId: string;
  quantity: number;
  qualityTier: number;
  storageFlag: "normal" | "backlog";
};
```

工厂状态负责表达：

- 工业路线
- 工厂等级
- 当前产能
- 用工成本
- 是否已安排生产

库存状态负责表达：

- 产出商品
- 当前库存
- 是否被压仓库

## 5. 阶段工作区

### 5.1 DecisionPhaseWorkspace

```ts
type DecisionPhaseWorkspace = {
  phase: "decision";
  availableActionsByPlayer: Record<PlayerId, {
    factoryActions: FactoryActionOption[];
    domesticMarketActions: DomesticMarketActionOption[];
    governmentActions: GovernmentActionOption[];
  }>;
  submittedPlayerIds: PlayerId[];
};
```

这里保存的是：

- 可执行工厂动作
- 可执行国民消费动作
- 可执行政府财政动作
- 已提交玩家列表

### 5.2 MarketPhaseWorkspace

```ts
type MarketPhaseWorkspace = {
  phase: "market";
  saleOptionsByPlayer: Record<PlayerId, {
    sellableInventory: InventoryState[];
    domesticMarketCapacity: number;
    overseasMarketCapacity: number;
    priceHints: PriceHint[];
  }>;
  submittedPlayerIds: PlayerId[];
};
```

这里保存的是：

- 可卖库存
- 国内市场承接能力
- 海外市场承接能力
- 价格预估与市场提示

### 5.3 SettlementPhaseWorkspace

```ts
type SettlementPhaseWorkspace = {
  phase: "settlement";
  settlementByPlayer: Record<PlayerId, {
    domesticSalesRevenue: number;
    overseasSalesRevenue: number;
    nationalIncome: number;
    budgetAllocation: {
      domesticMarket: number;
      factory: number;
      governmentFiscal: number;
    };
    nextRatio: {
      domesticMarket: number;
      factory: number;
      governmentFiscal: number;
    };
  }>;
};
```

这里保存的是阶段摘要，前端只读展示，不允许编辑。

## 6. 阶段输入结构

### 6.1 国家决策提交

```ts
type DecisionSubmission = {
  factoryPlan: {
    productionOrders: ProductionOrder[];
    expansionOrders: ExpansionOrder[];
    upgradeOrders: UpgradeOrder[];
    newFactoryOrders: NewFactoryOrder[];
  };
  domesticMarketPlan: {
    domesticMarketActions: DomesticMarketActionSelection[];
  };
  governmentPlan: {
    pointPurchases: PointPurchase[];
    strategySelections: GovernmentActionSelection[];
  };
};
```

### 6.2 市场出售提交

```ts
type MarketSubmission = {
  saleOrders: SaleOrder[];
};
```

财政结算阶段没有玩家提交结构。

## 7. 状态流转

### 7.1 回合开始

- 进入 `decision`
- 装载三个预算池、点数余额、工厂与库存状态
- 生成当前可执行动作

### 7.2 国家决策结算

- 校验三池余额
- 扣减本回合决策花费
- 更新工厂、消费能力、科技策略、军事策略、比例修正
- 生成市场出售阶段所需派生状态

### 7.3 市场出售结算

- 根据卖向与卖量分配库存
- 计算国内销售额与海外销售额
- 未卖出部分保留在库存中并标记是否压仓库
- 汇总国家收入

### 7.4 财政结算

- 按当前 `incomeAllocationRatio` 对 `nationalIncome` 重新分账
- 更新三个预算池
- 累加 `cumulativeNationalIncome`
- 清空回合临时收入字段
- 进入下一回合 `decision` 或结束游戏

## 8. 排名与阶段摘要

### 排名主口径

排行榜与最终结算都使用：

- `cumulativeNationalIncome`

### 阶段摘要必须展示

- 本回合国内销售额
- 本回合海外销售额
- 本回合国家收入
- 三个预算池各自分到多少
- 当前收入分配比例
- 本回合执行的关键政府策略

## 9. 恢复要求

刷新恢复至少需要保证：

- 回合数正确
- 阶段正确
- 三个预算池正确
- 点数正确
- 工厂、库存、比例状态正确
- 当前工作区可继续提交或继续查看结算

这意味着上面定义的快照字段必须全部可持久化，不能依赖仅存在于内存中的派生状态。
