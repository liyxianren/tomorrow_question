# 当前系统说明

> 文档定位：描述当前 Tomorrow Question 已经形成的系统形态。  
> 用途：客户沟通、工程接手、验收前快速统一口径。  
> 不替代用户需求源；需求源以《用户原始需求：核心机制》、会后经济机制纪要、第一阶段市场与生产机制文档为当前 2.0 迁移优先依据；旧确认文档保留为历史需求源。


## 0. 2026-04-27 会后状态说明

本文件描述的是当前已经能运行和演示的系统形态，不等于核心经济机制已经验收通过。

2026-04-27 会议确认：当前 Demo 的工业区 / 市场经济模型与客户预期存在根本差异，需要按 [2026-04-27 经济机制重构会议纪要](./2026-04-27-经济机制重构会议纪要.md) 和 [目标经济模型：供给 - 需求 - 价格机制](./目标经济模型-供需价格机制.md) 重构。

因此当前系统需要拆成两层理解：

1. `可保留的运行框架`：首页、大厅、房间、选国、Ready、AI 补位、主地图、三阶段外壳、议会厅 / 军事要塞 / 研究院等外围模块。
2. `必须重构的核心经济模型`：工业区生产、市民广场 / 市场需求、贸易港价格、财政结算中的国家收入来源。

## 1. 当前系统总览

Tomorrow Question 当前是一款 `5` 人固定国家、`15` 回合、同步提交、统一结算的网页策略游戏。

玩家扮演：

- 英国
- 法国
- 普鲁士
- 奥地利
- 俄罗斯

核心循环为：

```text
国家决策 -> 市场出售 -> 财政结算
```

胜负目标为：

```text
15 回合累计国家收入最高者获胜
```

## 2. 入局链路

当前入局链路已经形成：

1. 首页说明游戏定位与流程
2. 大厅确认本机身份 / 昵称
3. 创建房间或输入房间码加入
4. 房间内选择国家
5. 玩家 Ready
6. 房主可用 AI 补满空位
7. 满足条件后自动开局

开局条件：

- 5 个席位满员
- 每个席位都已选择国家
- 所有玩家 / AI 都已 Ready

## 3. 三个预算池

当前系统围绕三个独立预算池展开：

1. `国内消费市场`
   - 承担内需、国内承接能力、国内价格相关动作
2. `工厂`
   - 承担生产、扩产、升级、新建工厂等工业动作
3. `政府财政`
   - 承担科技点购买、军事/政策/国家结构变化相关动作

这三个池不是共享钱包，不能随意互相挪用。

## 4. 国家决策阶段

国家决策阶段当前分为五个真实游戏入口：

1. `工业区`
   - 工厂预算
   - 产线排程
   - 建设改造
   - 工业研究
2. `市民广场`
   - 国内预算
   - 民生政策
   - 国内市场容量 / 国内消费能力相关动作
3. `议会厅`
   - 政府财政
   - 科技点拨款 / 购买
   - 政策与国家能力相关动作
4. `军事要塞`
   - 军事点
   - 舰队
   - 海外桥接
   - 建交与海外区域准入
5. `研究院`
   - 科技点
   - 工业 / 市民 / 政府 / 军事方向天赋树

## 5. 市场出售阶段

市场出售阶段当前以 `贸易港` 作为主入口。

玩家在这一阶段安排库存商品销售：

- 国内市场
- 欧洲等开放海外市场
- 亚太等特许权市场

系统展示：

- 库存
- 参考价
- 国内承接
- 海外承接
- 预计收入

## 6. 财政结算阶段

财政结算阶段是只读系统阶段，用于展示：

- 本回合国内销售额
- 本回合海外销售额
- 本回合国家收入
- 累计国家收入
- 下一轮收入分配比例
- 三池重新分配结果

结算完成后，系统进入下一回合。

## 7. 实时与恢复

当前系统包含以下运行时能力：

- Socket 连接状态展示
- 房间状态同步
- 阶段提交状态同步
- Bot 自动提交
- 刷新恢复 / 上次会话恢复
- 结算倒计时
