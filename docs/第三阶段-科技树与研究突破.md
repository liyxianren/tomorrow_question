# 第三阶段：科技树与研究突破

> 文档定位：第三阶段需求分析与实施设计。以用户原始需求 §4 科技树框架为唯一需求源。
> 适用范围：研究设施建设、研究进展累积、科技突破掷骰子、科技解锁生产方式门控。

## 一、用户需求骨架

### 1.1 核心闭环

```
财政池 → 建研究设施 → 每回合 +1 进展 → 攒够阈值 → 掷骰子突破 → 解锁科技 → 解锁生产方式
```

### 1.2 三支科技链

| 链条 | 科技序列 | 终点解锁 |
|------|---------|---------|
| 电气 | 莱顿瓶(3) → 伏打电堆(4) → 发电(5) | 电气化生产 |
| 机械 | 珍妮纺纱机(3) → 车床(4) | 机械生产 |
| 蒸汽 | 贝塞麦炼钢(3) → 瓦特蒸汽机(4) → 内燃机(5) | 蒸汽生产 |

括号内为突破阈值（所需最少骰子点数）。

### 1.3 用户数值（问题与修正）

| 参数 | 用户原值 | 问题 | 我们的修正 | 理由 |
|------|---------|------|-----------|------|
| 突破骰子 | 未指定 | D6 下阈值 5 只有 33% 成功率，运气差卡 6-8 回合 | **D10** | 进展积攒是主门控，骰子只加 ±1 回合噪音 |
| 研究设施成本 | 未定 | — | **12 预算**（从财政池扣） | ≈2 回合储蓄，与其他政府行动价格持平 |
| 非首个研究者 | 未说明 | — | **直解：达到原阈值进展，无骰子** | 鼓励追赶，避免先发垄断；规则更直观 |
| 保底机制 | — | D10 下仍有 40% 连续失败可能 | **软保底：每次失败阈值 -1，成功后重置** | 最差 3 次必过，保留运气成分 |

### 1.4 与一二阶段的结藕边界

- **不碰 Phase 1 的 `routeUnlocks` 消费逻辑**：`decision.py` 中建厂/升级前的 `route_unlocks` 检查保持原样。Phase 3 只补"科技从哪来"。
- **不碰 Phase 2 的政府系统**：改革/政策/思潮完全不依赖科技树。唯一重叠是两者都消耗财政池（`governmentFiscal`），这在结算阶段已有规则处理。
- **复用现有 `/setting` 面板**：科技树参数（设施成本、科技阈值、D10 面数）追加到 `/setting` 配置页。

## 三、设计规范

### 3.1 数据模型

#### 3.1.1 配置（technology.json）

```jsonc
{
  // === 保留不动（Phase 1 消费） ===
  "routeUnlocks": {
    "mechanized": "spinning_jenny",       // 机械生产 ← 珍妮纺纱机
    "steam": "watt_engine",                // 蒸汽生产 ← 瓦特蒸汽机
    "electrified": "power_generation"      // 电气化生产 ← 发电
  },

  // === 新增 ===
  "researchFacilityCost": 12,             // 建设一个研究设施消耗的财政池预算
  "researchFacilityProgressPerTurn": 1,   // 每设施每回合产生的进展
  "breakthroughDieSides": 10,             // 突破骰子面数（D10）

  // === 三支科技链（替换旧 techTree） ===
  "chains": {
    "electrical": {
      "label": "电气链",
      "techs": [
        { "id": "leyden_jar",       "label": "莱顿瓶",     "threshold": 3 },
        { "id": "voltaic_pile",     "label": "伏打电堆",   "threshold": 4 },
        { "id": "power_generation", "label": "发电",       "threshold": 5 }
      ]
    },
    "mechanical": {
      "label": "机械链",
      "techs": [
        { "id": "spinning_jenny", "label": "珍妮纺纱机", "threshold": 3 },
        { "id": "lathe",          "label": "车床",       "threshold": 4 }
      ]
    },
    "steam": {
      "label": "蒸汽链",
      "techs": [
        { "id": "bessemer_process",  "label": "贝塞麦炼钢", "threshold": 3 },
        { "id": "watt_engine",       "label": "瓦特蒸汽机", "threshold": 4 },
        { "id": "combustion_engine", "label": "内燃机",     "threshold": 5 }
      ]
    }
  }
}
```

**关键设计决策**：

- `routeUnlocks` 映射：机械生产由链条**第一级**解锁（珍妮纺纱机），蒸汽生产由链条**第二级**解锁（瓦特蒸汽机），电气化生产由链条**第三级**解锁（发电）。这保证了三生产方式的解锁节奏有梯度——机械最早、蒸汽中期、电气化晚期。
- 链条内科技**顺序依赖**：必须先解锁莱顿瓶才能研究伏打电堆。三条链**相互独立**。
- 旧 `techTree` 中的非生产方式科技（纺织技术/矿业开发/茶道工艺/行政改革等）——这些是解锁商品和行动的辅助科技，不在三支链覆盖范围内。**本次不迁移，直接废弃**。如后续需要"解锁商品"的科技系统，另起炉灶。

#### 3.1.2 运行时状态（PlayerState 新增字段）

```python
@dataclass
class PlayerState:
    # ... 已有字段 ...
    active_research: Optional[str] = None          # 当前研究目标 tech_id
    research_progress: dict[str, int] = field(     # 每科技累积进展
        default_factory=dict

    breakthrough_attempts: dict[str, int] = field( # 软保底累计失败次数
        default_factory=dict

```

**`research_facilities` 语义保持不变**：`{"academy": N}`，N 为设施总数。`countries.json` 初始值保留（英/普 1，法/奥/俄 0）。

#### 3.1.3 科技解锁判定

```python
def is_tech_unlocked(player: PlayerState, tech_id: str) -> bool:
    return tech_id in player.unlocked_techs
```

**替换**旧 `is_tech_researchable`。Phase 1 的 `route_unlocks` 检查（`required_tech not in player_state.unlocked_techs`）不需要改动——它只读 `unlocked_techs` 列表，Phase 3 写入同一个列表。

### 3.2 核心流程

```
决策阶段                          结算阶段
────────                          ────────
玩家选择"建研究设施"  ─────────→  扣预算，+1 academy
玩家选择"研究目标"    ─────────→  写 active_research

                                  ┌─ 进展累积 ─────────────┐
                                  │ progress += facilities  │
                                  │ 如果 progress ≥ threshold│
                                  │   → 掷 D10              │
                                  │     成功 → 解锁科技      │
                                  │     失败 → threshold-1   │
                                  └────────────────────────┘
```

#### 3.2.1 决策阶段：建设研究设施

复用现有 `expand_research` 行动：
- 消耗 `governmentFiscal` 池 **12** 预算（原 10）
- `research_facilities["academy"] += 1`
- 每回合限 1 次

> **设计理由**：建研究设施是永久投资，收益线性累积。12 预算约等于数回合政府储蓄，需要限制每回合 1 次，防止开局砸 3 个设施直接 roll 穿一条链。

#### 3.2.2 决策阶段：设置研究目标

新增决策行动 `set_research_target`：
- 消耗：0 预算（免费切换）
- 参数：`tech_id`
- 校验：
  - 科技存在于三支链中
  - 该链条前一科技已解锁（或该科技为链条第一级）
  - 自身未被解锁
- 写入 `active_research`

切换目标时：**不丢失旧目标的累积进展**。玩家可以在多条链之间轮换研究。

#### 3.2.3 结算阶段：研究进展与突破

在 `settlement.py` 结算阶段，在思潮递进之后、收入分配之前，插入：

```python
def _apply_phase3_research_progress(player: PlayerState, balance):
    active = player.active_research
    if active is None:
        return
    
    tech = get_tech_by_id(active, balance)
    if tech is None or active in player.unlocked_techs:
        return
    
    # 1. 进展累积
    facilities = sum(player.research_facilities.values())
    progress = player.research_progress.get(active, 0) + facilities
    player.research_progress[active] = progress
    
    # 2. 判断是否达到阈值
    effective_threshold = max(1, tech.threshold - 
        player.breakthrough_attempts.get(active, 0))
    
    if progress < effective_threshold:
        return  # 不够，下回合继续
    
    # 3. 检查是否可直解（已被其他国家发现）
    if is_tech_discovered(active, balance):  # 任一国家已解锁
        if progress >= tech.threshold:
            # 直解：消耗原阈值进展
            player.research_progress[active] = progress - tech.threshold
            player.unlocked_techs.append(active)
            player.breakthrough_attempts.pop(active, None)
            return
        # 后发研究者不再掷骰，继续累积到原阈值
        player.breakthrough_attempts.pop(active, None)
        return
    
    # 4. 首个发现者突破掷骰子
    roll = random.randint(1, balance.technology.breakthrough_die_sides)
    if roll >= effective_threshold:
        # 成功
        player.unlocked_techs.append(active)
        player.research_progress[active] = 0
        player.breakthrough_attempts.pop(active, None)
    else:
        # 失败：软保底
        player.breakthrough_attempts[active] = (
            player.breakthrough_attempts.get(active, 0) + 1
        )
        # 进展保留，下回合阈值更低
```

### 3.3 边界条件

| 场景 | 处理 |
|------|------|
| 未设研究目标 | 不累积进展，不掷骰子 |
| 目标科技已被解锁 | 跳过，进展归零 |
| 设施数为 0 | 不累积进展（无法研究） |
| 多条链同时研究 | 不支持——只有 1 个 `active_research`。这是设计意图：15 回合你只能选 1-2 条路线 |
| 切换目标 | 旧目标进展保留，新目标从已有进展继续 |
| 直解时进展不够原阈值 | 继续正常累积，不再掷骰；后发研究者达到原阈值就直接解锁 |
| 保底降到 1 | `effective_threshold = max(1, ...)` 保证必然成功 |

### 3.4 数值平衡验证

**场景：英国（初始 1 academy，建到 3 个）**

| 回合 | 行动 | 积累进展 | 目标 |
|------|------|---------|------|
| 1 | 建 academy #2 (-12) | 0 | 珍妮纺纱机 |
| 2 | 建 academy #3 (-12) | 2 | — |
| 3 | 设目标：珍妮纺纱机 | 5 (≥3) → 掷 D10 (80%) | ✅ 大概率解锁 |
| 4 | 设目标：车床 | 3 | — |
| 5 | — | 6 (≥4) → 掷 D10 (70%) | ✅ 大概率解锁 |

→ 机械链：5 回合走完。蒸汽链（7 总阈值）≈9 回合。电气链（12 总阈值）≈13 回合。

符合预期：15 回合只够走 1.5-2 条链。

## 六、风险与注意事项

1. **旧 `techTree` 废弃影响面**：目前 workspace 中 `_build_tech_tree` 只用于决策阶段展示，无结算阶段消费。废弃不会破坏现有游戏流程。但前端 `Phase1ProductionPanel` 可能依赖 `techTree` 的 `canResearch` 字段来决定建厂/升级按钮是否可点——需要确认前端逻辑后适配。

2. **`expand_research` 成本改动**：如果已有 saved games 中玩家已购买设施，不回溯调整。新游戏生效。

3. **随机数测试**：突破掷骰子需要 mock `random.randint`。TDD 时先写无随机注入的纯逻辑测试，再加 mock 测试。

4. **直解的"已被发现"判定**：需要查询所有玩家（不仅仅是当前玩家）的 `unlocked_techs`。结算阶段已有 `snapshot.player_states` 遍历，可以直接取并集。

5. **不迁移旧 `techTree` 非生产方式科技**：纺织技术/矿业开发/茶道工艺等旧科技直接废弃。这些科技解锁的商品（棉花/矿物/茶叶）在当前经济系统中如果被引用，需要在 Phase 1 配置中直接设为初始可用（不依赖科技解锁）。这需要在 Task 1 中一并检查。
