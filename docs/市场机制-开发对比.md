> 文档定位：对比已完成的 Phase1 市场机制开发与用户原始需求的匹配度，标注差异与待补项。
> 适用对象：开发者、验收者
> 生成时间：2026-04-28

---

## 一、当前实现的架构

代码中存在**双路径**运行：

```
提交时判断 payload 中是否有 phase1Production 字段
├── 有 → Phase1 新路径（统一商品模型）
│    ├── _apply_phase1_production_plan()   ← 原材料→商品，建/升级产能
│    ├── _apply_phase1_market()            ← 供需定价，国内+海外投放
│    └── settlement                         ← 5:3:2 收入分配
│
└── 无 → Legacy 旧路径（多具体商品模型）
     ├── _apply_factory_plan()             ← goodsId 生产
     ├── legacy market                      ← 参考价+阈值
     └── settlement                         ← 3:3:4 权重
```

**核心纯计算模块**（与 Flask/会话无关）：`backend/app/modules/rules/phase1_economy.py`

### 2.2 价格形成机制

| 规则 | 用户原始需求 | 代码实现 | 匹配 |
|------|------------|---------|------|
| 均衡价格 | 消费池资金 ÷ 需求 | `calculate_equilibrium_price(consumption_pool, demand)` | ✅ |
| 消费池时点 | "本回合消费池资金" | 市场阶段开始时（`player_state.budget_pools["domesticMarket"]`） | ✅ |
| 短缺涨价 | 短缺率 = (需求-供给)/需求，价格上涨同等百分比 | `shortage_rate = (demand - supply)/demand`，`price = equilibrium × (1 + shortage_rate)` | ✅ |
| 过剩降价 | 过剩率 = (供给-需求)/需求，价格下降同等百分比 | `surplus_rate = (supply - demand)/demand`，`price = equilibrium × (1 - surplus_rate)` | ✅ |
| 价格下限 | 原始文档未提及 | `DEFAULT_MINIMUM_DOMESTIC_PRICE = 1` | ✅ 新增保护 |
| 成交量 | 供给<需求时按供给，供给>需求时按需求 | `sold = min(supply, demand)` | ✅ |

**结论：✅ 公式完全一致。** 价格下限是工程保护，防止极端过剩导致 0 或负价格。

## 四、我的理解

根据文档整理和代码阅读，我的理解是：

**你已经完成了第一阶段的核心交付——市场机制。** 代码实现了用户原始需求中最关键的三个闭环：

1. **生产闭环**：原材料 → 生产方式选择 → 统一商品库存
2. **定价闭环**：消费池 → 需求 → 均衡价格 → 供需波动 → 最终价格
3. **分配闭环**：销售收入 → 5:3:2 → 消费池/投资池/财政池

这三个闭环完整实现了用户要求的"宏观经济模型"——不是多商品库存小游戏，而是供给/需求/价格/分配的完整链路。

**但有一个关键问题：** 当前代码是双路径运行的，Phase1 新路径（统一商品）和 Legacy 旧路径（多具体商品）共存。你需要决定是否切掉旧路径，让全系统只走 Phase1 模型。

**数值层面**：产能建设价格（20/40/60/80）和半价升级规则需要在 `production.json` 中对齐。

**下一阶段的工程问题**：外部市场的独立供需定价、军事封锁对贸易的实际影响、政治改革与经济体的联动闭环——这些是 Phase1 文档中明确标注"后续阶段"的内容，尚未实现。
