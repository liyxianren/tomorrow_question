## 1. 当前问题

游戏核心循环（决策→出售→结算）已跑通，但可玩性不足：

1. **无决策深度** — 所有商品/策略一开始全部可用，最优策略是"把预算花光"
2. **每回合决策相同** — 无随机事件，无需适应变化 ✅ 已通过 F1 解决
3. **玩家之间零互动** — 各玩各的单机体验
4. **价格永远不变** — 无供需博弈，最优策略一成不变
5. **国家手感雷同** — 只有初始数值差异，几回合后趋同

## 3. F5：渐进解锁科技树（瘟疫公司风格）

### 3.1 设计目标

改造核心机制：**商品和策略不再全部可用，需要花科技点逐步解锁**。

- 线性链式解锁（类似瘟疫公司）
- 每国只有 2-3 种初始可生产商品
- 策略也分层解锁（基础免费，高级需科技前置）
- 15 回合内不可能全部解锁 → **必须选择发展路线**

### 3.2 每国初始可用商品

| 国家 | 初始商品 | 初始路线 | 设计理由 |
|------|---------|---------|---------|
| 英国 | 煤炭、棉花 | handicraft=2 | 工业+纺织帝国 |
| 法国 | 粮食、茶叶 | handicraft=1 | 农业+奢侈品 |
| 普鲁士 | 煤炭、钢铁 | handicraft=1, mechanized=1 | 重工业起步 |
| 奥地利 | 粮食、矿产 | handicraft=1 | 农矿国家 |
| 俄罗斯 | 粮食、煤炭 | handicraft=2 | 资源大国 |

### 3.3 科技树结构

```
═══ 工业科技链（花工厂预算解锁） ═══

初始商品（免费，每国 2-3 种）
│
├─ [纺织技术] 花 6 工厂预算 → 解锁 棉花
├─ [茶道工艺] 花 6 工厂预算 → 解锁 茶叶
├─ [矿业开发] 花 6 工厂预算 → 解锁 矿产
│
└─ [珍妮纺织机] 花 10 工厂预算 → 解锁 机械化路线（钢铁、丝绸）
     │
     └─ [蒸汽引擎] 花 18 工厂预算 → 解锁 蒸汽路线（石油）
          │
          └─ [电气化] 花 30 工厂预算 → 解锁 电气路线（橡胶）

═══ 政策科技链（花政府预算解锁） ═══

├─ [行政改革] 花 10 政府预算 → 解锁 扩展航线(10)、产业政策(12)
│    │
│    └─ [帝国体制] 花 16 政府预算 → 解锁 远征军筹建(20)、海军现代化(14)
│         │
│         └─ [全球霸权] 花 24 政府预算 → 解锁 殖民特许(34)

═══ 消费科技链（花国内预算解锁） ═══

├─ [市场经济] 花 6 国内预算 → 解锁 消费补贴(8)、进口替代(9)
│    │
│    └─ [消费社会] 花 10 国内预算 → 解锁 公共工程(10)、奢侈品推广(12)
```

### 3.4 基础层（免费可用，无需解锁）

**策略**：贸易协定(6)、征兵动员(8)、内需刺激(10)
**国内动作**：博览会(5)、乡村开发(6)
**商品**：每国初始 2-3 种

### 3.5 预算总量控制

**核心设计**：科技解锁直接花对应预算池的钱，和生产/策略/消费竞争同一个池子。
- 工业科技和生产抢工厂预算 → 投科技就少生产
- 政策科技和策略执行抢政府预算 → 投科技就少执行策略
- 消费科技和消费动作抢国内预算 → 投科技就少铺承接力

**全部解锁所需预算**：
- 工业链：6+6+6+10+18+30 = **76 工厂预算**
- 政策链：10+16+24 = **50 政府预算**
- 消费链：6+10 = **16 国内预算**

**15 回合预算总量推算**（以英国为例，初始工厂预算12，每回合收入分配约25%到工厂）：
- 15 回合工厂预算总计约 **50-80**（取决于收入增长）
- 如果全投科技，几乎没有预算生产 → 必须平衡

### 3.6 15 回合节奏推演

| 回合 | 典型进度 | 解锁内容 | 可用操作 |
|------|---------|---------|---------|
| 1-2 | 攒预算 | 无 | 2-3 种初始商品 + 3 基础策略 + 2 基础动作 |
| 3-4 | 解锁 1 级科技 | 纺织/矿业(工厂6) 或 市场经济(国内6) | +1-2 种商品 或 +2 中级动作 |
| 5-6 | 解锁 2 级科技 | 珍妮纺织机(工厂10) 或 行政改革(政府10) | 机械化路线 或 中级策略 |
| 7-9 | 解锁 3 级科技 | 蒸汽引擎(工厂18) 或 帝国体制(政府16) | 蒸汽路线 或 高级策略 |
| 10-12 | 解锁 4 级科技 | 电气化(工厂30) 或 全球霸权(政府24) | 电气路线 或 终极策略 |
| 13-15 | 终盘博弈 | 预算用于生产+出售最大化收入 | 已解锁内容全力运转 |

**文件**：`backend/app/modules/game_state/factory_economy.py`

修改 `goods_locked_reason()` 函数（第 68 行）：

```python
def goods_locked_reason(player, route_id, goods_id=None) -> str | None:
    balance = get_balance_config()
    
    # 1. 检查是否是初始可用商品
    country_config = balance.countries.get(player.country.value)
    if country_config and goods_id and goods_id in country_config.initial_goods:
        # 初始商品只检查产能
        if current_route_capacity(player, route_id) > 0:
            return None
        return f"需要{get_route_label(route_id)}产能"
    
    # 2. 检查是否通过科技解锁
    if goods_id:
        unlocked = _is_goods_unlocked_by_tech(player, goods_id, balance)
        if not unlocked:
            tech_label = _find_required_tech_label(goods_id, balance)
            return f"需要研究「{tech_label}」"
    
    # 3. 检查产能
    if current_route_capacity(player, route_id) <= 0:
        return f"需要{get_route_label(route_id)}产能"
    return None

def _is_goods_unlocked_by_tech(player, goods_id, balance) -> bool:
    for tech in balance.technology.tech_tree.values():
        if goods_id in tech.unlocks_goods:
            return tech.tech_id in player.unlocked_techs
    return True  # 如果没有科技要求，默认可用

def _find_required_tech_label(goods_id, balance) -> str:
    for tech in balance.technology.tech_tree.values():
        if goods_id in tech.unlocks_goods:
            return tech.label
    return "未知科技"
```

#### 步骤 7：科技研究处理

```python
def _apply_tech_research(player_state, tech_selections, balance):
    """处理玩家的科技研究选择"""
    if not tech_selections:
        return
    for selection in tech_selections:
        tech_id = selection.get("techId")
        if not tech_id:
            continue
        tech = balance.technology.tech_tree.get(tech_id)
        if tech is None:
            continue
        # 已解锁
        if tech_id in player_state.unlocked_techs:
            continue
        # 检查前置
        if not all(p in player_state.unlocked_techs for p in tech.prerequisites):
            continue
        # 检查科技点
        if player_state.tech_points < tech.cost:
            continue
        # 扣除科技点，解锁
        player_state.tech_points -= tech.cost
        player_state.unlocked_techs.append(tech_id)
```
