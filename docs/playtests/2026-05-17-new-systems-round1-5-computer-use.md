# 2026-05-17 新工厂 / 政府 / 市场政策 1-5 回合 Computer Use 玩家测试

> 本轮按真实玩家视角手工游玩，重点检查最近改动的新工厂建设 / 产业升级、政府政策、市场政策，以及这些内容在预算、预览、提交、结算中的一致性。

## 结论摘要

- 1-5 回合可以通过保守玩法跑完，但新内容的核心链路仍有阻断级问题。
- `产业升级` 在第 5 回合解锁机械化后，UI 显示机械化产能可本回合生产，但提交校验仍按机械化容量 `0` 拒绝，导致玩家必须放弃本回合机械化生产才能继续。
- 政府大厅里的三张市场政策会消耗行政力，但市场页预览基本没有反映这些政策，玩家无法确认市场政策是否生效。
- 工厂预算、市场预览、收入分配结算存在多处口径不一致，尤其是工厂面板内的预算分母会随建设成本缩水。

## 详细问题

### 1. 复杂新内容组合提交失败

- 回合：对局 A，第 1 回合
- 操作组合：
  - 工厂增加：手工业 `2` 次
  - 政府市场政策：`市场补贴`、`价格管制`、`贸易促进` 全选
  - 军事：征募陆军 `1` 次
  - 研究：`珍妮纺纱机`
- 期望：如果资源 / 行政力合法，应正常提交；如果不合法，应给出具体资源或规则错误。
- 实际：点击提交后显示 `后端服务不可用，请确认本地 API 已启动。`
- 备注：当时后端健康检查正常，且该回合没有看到有效玩家回合输入落库。玩家角度看这是阻断级错误，但本报告不判断根因是网络、前端请求构造还是后端异常。

### 2. 产业升级后的机械化产能不能真正提交

- 回合：对局 B，第 5 回合
- 前置：`珍妮纺纱机` 已解锁，机械化路线在工厂页开放。
- 操作：
  1. 选择 `产业升级：机械化`。
  2. UI 显示手工业产能 `9 -> 8`，机械化产能 `0 -> 1`。
  3. 玩家手动把 1 点原料从手工业挪到机械化生产。
- 期望：因为 UI 文案说产业升级会把低级产能立即转为高级产能，并且本回合可生产，所以机械化分配 `1` 应该可提交。
- 实际：提交按钮被禁用，校验文本显示 `{{mode}} 分配 1 超出容量 0`。
- 临时绕过：把机械化生产改回 `0`，只用手工业生产，才能继续完成第 5 回合。
- 影响：这是新工厂逻辑的核心路径，玩家第一次解锁机械化就会遇到“UI 说可用、规则说不可用”的矛盾。

### 3. 市场政策选择后，市场预览没有同步

- 回合：对局 B，第 2-4 回合
- 观察：
  - 第 2 回合选 `市场补贴`，行政力从 `3 / 3` 变为 `2 / 3`。
  - 第 3 回合选 `价格管制`，行政力从 `3 / 3` 变为 `2 / 3`。
  - 第 4 回合选 `贸易促进`，行政力从 `3 / 3` 变为 `2 / 3`。
- 期望：市场页或紧凑预览应显示本轮政府市场政策对价格、需求、投放上限、海外容量的影响。
- 实际：市场预览仍出现 `政府调整 暂不可用`、`暂无市场调节` 等状态。
- 影响：玩家无法知道市场政策是否已经生效，只能在售卖结果里猜。

### 4. 市场预览数值口径冲突

- 回合：对局 B，第 3 回合
- 操作：选择 `价格管制`。
- 观察：
  - 政策文案表达的是国内价格调整。
  - 预览里同时出现顶部均衡价 `7`、详情均衡参考价 `5`、`市场需求 +2` 等口径。
- 期望：价格管制到底影响价格、需求、还是投放上限，需要在同一个预览体系里一致表达。
- 实际：玩家无法从预览判断最终售卖时会使用哪个数字。

### 5. 工厂预算分母在面板内缩水

- 回合：对局 A 第 1 回合、对局 B 第 2 回合、第 5 回合
- 观察样例：
  - 对局 A 第 1 回合：顶部工厂预算 `3 / 14`，工厂概览显示 `3 / 11`；第二次工厂增加后顶部 `0 / 14`，概览 `0 / 8`。
  - 对局 B 第 2 回合：顶部 `7 / 18`，工厂概览 `7 / 15`。
  - 对局 B 第 5 回合：选择机械化产业升级后顶部 `61 / 75`，工厂概览 `61 / 63`；再选择机械化工厂增加后顶部 `36 / 75`，工厂概览 `36 / 38`。
- 期望：如果显示的是“剩余 / 总量”，分母应该保持同一个总预算；如果显示的是“剩余 / 可用于生产”，文案必须说明。
- 实际：顶部与工厂面板分母不同，且没有解释，玩家会理解为预算总量被改写。

### 6. 结算比例显示与实际分配不一致

- 回合：对局 B，第 1-5 回合
- 观察样例：
  - 第 1 回合售卖收入 `24`，结算页显示比例 `4 / 3 / 3`，但民间购买力 `0`、工厂 `12`、政府财政 `12`。
  - 第 3 回合售卖收入 `79`，结算页显示比例 `4 / 3 / 3`，但民间购买力 `0`、工厂 `39`、政府财政 `40`。
  - 第 5 回合售卖收入 `24`，结算页仍显示比例 `4 / 3 / 3`，但民间购买力 `0`、工厂 `12`、政府财政 `12`。
- 期望：如果 `4 / 3 / 3` 是本轮收入分配比例，民间购买力不应长期为 `0`；如果民间购买力代表另一个模型，需要在结算页解释。
- 实际：玩家看到比例和分配结果对不上。

### 7. 政府市场政策的选择状态不清楚

- 回合：对局 B，第 2-4 回合
- 观察：
  - 选择市场政策后行政力会减少。
  - 但按钮仍显示 `选择：市场补贴` / `选择：价格管制` / `选择：贸易促进`。
  - `本轮已选政策` 区域仍显示 `本轮暂未选择政策`。
- 期望：已选市场政策应该有明确选中态，并出现在本轮选择摘要中，或有一个专门的 `本轮市场政策` 摘要。
- 实际：玩家不知道自己是否已经选上，只能从行政力变化反推。

### 8. 征募陆军的政府财政计划成本没有进入顶部预算

- 回合：对局 A，第 1 回合
- 操作：选择 `征募陆军`。
- 期望：如果征募消耗政府财政，顶部政府财政预算应同步变动；如果不是当前阶段预算支出，应明确说明。
- 实际：选择征募后顶部政府财政仍显示 `18 / 18`。

### 9. 工厂前置条件说明有改善，但仍需要和校验统一

- 回合：对局 B，第 1-5 回合
- 正向观察：锁定路线现在会说清楚需要的科技和上游产能，例如机械化路线需要 `珍妮纺纱机` 与手工业产能。
- 剩余问题：第 5 回合解锁机械化后，文案说产业升级可以本回合转产，但提交校验仍按机械化容量 `0` 拒绝。

## 回合记录

### 对局 A：第 1 回合压力路径

- 国家：英国
- 起始资源：民间购买力 `24`、工厂 `14`、政府财政 `18`
- 操作：连续选择两个手工业工厂增加、三个市场政策、一次征募陆军、研究 `珍妮纺纱机`

### 对局 B：第 1 回合

- 操作：保守玩法，只做默认工厂生产，不选政府 / 军事 / 研究
- 售卖：国内市场售出 `8`，单价 `3`，收入 `24`
- 结算：显示比例 `4 / 3 / 3`，但分配为民间购买力 `0`、工厂 `12`、政府财政 `12`
- 下一回合资源：民间购买力 `24`、工厂 `18`、政府财政 `38`

### 对局 B：第 2 回合

- 工厂：选择 `工厂增加：手工业` 1 次，顶部工厂预算 `7 / 18`，工厂概览 `7 / 15`
- 政府：选择 `市场补贴`，行政力 `3 / 3 -> 2 / 3`
- 市场预览：仍显示没有政府调整 / 暂无市场调节
- 研究：选择 `珍妮纺纱机`
- 售卖：亚洲 `6` 件、国内 `2` 件，总收入 `41`

### 对局 B：第 3 回合

- 工厂：上一回合手工业新增产能已生效，手工业产能到 `9`
- 政府：选择 `价格管制`，行政力 `3 / 3 -> 2 / 3`
- 市场预览：顶部均衡价、详情参考价、需求变化口径冲突
- 售卖：国内售出 `8`，亚洲售出 `1`，总收入 `79`
- 结算：比例仍显示 `4 / 3 / 3`，民间购买力分配仍为 `0`

### 对局 B：第 4 回合

- 工厂：机械化仍锁定，`珍妮纺纱机` 进度显示 `2 / 3`
- 政府：选择 `贸易促进`，行政力 `3 / 3 -> 2 / 3`
- 市场预览：仍显示 `暂无市场调节`，未清楚展示海外容量提升
- 售卖：亚洲售出 `6`，收入 `48`

### 对局 B：第 5 回合

- 前置：`珍妮纺纱机` 解锁，机械化路线开放
- 工厂：
  - 选择 `产业升级：机械化` 后，UI 显示手工业 `9 -> 8`、机械化 `0 -> 1`
  - 选择 `工厂增加：机械化` 后，显示机械化下回合产能 `+2`
  - 工厂预算顶部与面板概览继续不一致：例如顶部 `36 / 75`，工厂概览 `36 / 38`
- 阻断点：
  - 把 1 点原料分配到机械化后，提交校验报 `{{mode}} 分配 1 超出容量 0`
  - 改回机械化分配 `0` 后才能提交
- 研究：选择 `车床`
- 售卖：国内售出 `2`，单价 `12`，收入 `24`
- 结算：比例 `4 / 3 / 3`，民间购买力分配仍为 `0`

## 待确认的规则问题

- `4 / 3 / 3` 是否真的是本轮售卖收入分配？如果不是，结算页文案需要改成实际规则。
- 市场政策应该出现在 `本轮已选政策`，还是应该有独立的 `本轮市场政策` 摘要？目前两边都不清楚。
- `产业升级` 是否设计为本回合立即可生产？当前 UI 文案说可以，提交校验说不可以。
- 工厂预算面板里的分母是否想表达“扣除建设后的可生产预算”？如果是，需要改名；如果不是，就是数值 bug。

## 建议修复顺序

1. 先修 `产业升级` 后机械化生产被 `容量 0` 拦截的问题，并处理 `{{mode}}` 未插值。
2. 再查复杂新内容组合提交失败，保证非法计划返回具体校验错误，而不是 `后端服务不可用`。
3. 修市场政策预览：选择、行政力消耗、市场页预览、售卖结果必须能对上。
4. 修工厂预算分母和结算收入分配口径，避免玩家看到多个互相矛盾的总量。
5. 补充已选市场政策的明确选中态和摘要展示。
