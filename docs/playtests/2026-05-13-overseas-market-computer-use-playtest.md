## 主要问题

### P0：资源显示和后端真实值不一致

多回合出现 UI 顶部资源与后端快照不一致：

- 第 5 回合 UI 显示政府财政 `96`、军力 `2`，后端为政府财政 `67`、步兵 `0`、炮兵 `0`。
- 第 7 回合 UI 显示军事面板 `3/3`，后端实际军事点 `2`，选择 3 个步兵后提交被后端拒绝。
- 第 8 回合 UI 显示政府财政 `102`、军力 `2`，后端为政府财政 `86`、步兵 `1`。
- 第 10 回合 UI 显示军力 `4`、军事面板 `4/4`，后端实际军事点 `2`。

这不是纯展示问题。它会诱导玩家做出后端拒绝的计划，也会让玩家误判政府财政、军力、军事点是否足够。

建议：

- 所有阶段顶部资源、地图卡片、军事面板都只从同一个 workspace/state 派生。
- 提交前的前端校验必须使用与后端同源的 `availableMilitaryPoints` 和 `army`。
- 事件加成如果只存在于 workspace 预览，不要直接覆盖国家长期状态显示。

### P0：国家决策阶段可绕过未决策面板

从第 2 回合开始，政府政策和军事要塞仍显示“未决策”，但点击侧栏“确认提交”可以直接进入市场出售阶段。更严重的是，进入市场后资源池会变化，表现像某些默认或旧草稿决策被提交了。

例子：

- 第 2 回合决策 UI：民间购买力 21、工厂 16、政府财政 36。
- 直接提交后市场 UI：民间购买力 21、工厂 8、政府财政 15。

建议：

- 未完成必填面板时，侧栏确认提交应阻断，并列出缺失项。
- 如果允许“默认跳过”，按钮文案必须明确为“按默认方案提交”，并展示将要消耗的工厂/政府/军事点。
- 草稿应在回合切换时重置，避免前一回合选择被隐式带入。

### P1：海外争夺结算能工作，但价格表达不清

争夺成功时，结算确实把胜利价格 `+1` 加入了收入：

- 第 1 回合中东主价格 3，胜利后 8 件收入 32，即 4/件。
- 第 4 回合中东主价格 9，胜利后 7 件收入 70，即 10/件。
- 第 8 回合中东主价格 14，胜利后 2 件收入 30，即 15/件。

但区域卡片的大号“海外价格”仍显示和平外销价，胜利价只体现在预计总收入或小字里。玩家看到“价格 9”，结算却是“10/件”，容易误以为价格计算错了。

建议：

- 区域卡同时显示“保底价格”和“夺取成功价格”。
- 总收入旁标注公式：`2 件 × (14 + 争夺胜利 1) = 30`。
- 如果已投入兵力，卡片主价格可切换为“夺取成功预计价”，保底价降级为辅助信息。

### P1：政府财政/市场调节预算仍然会让人误读

当前 UI 的“政府财政”看起来仍然混合了基础政府财政和市场调节预算。第 1 回合后端预算池是：

```json
{
  "domesticMarket": 24,
  "factory": 14,
  "governmentFiscal": 10
}
```

但 UI 头部显示政府财政 34，并在政府面板里展示基础 10/10、市场调节 24/24。玩家会自然理解为政府真的有 34 点可自由支配，但实际市场调节动作与政府/军事/研究动作的预算语义不同。

建议：

- 头部不要只写“政府财政 34”，改成“政府财政 10 + 市场调节 24”。
- 市场调节区的成本消耗应明确写入哪个池，或者彻底把 `domesticMarket` 改名为“市场购买力/内需池”，避免被当作政府点数。

### P1：数值滚雪球过强，海外路线压倒其他系统

俄罗斯最终收入 453，第二名 139。这个差距不是由产能优势造成的，俄罗斯终局总产能只有 8，而 Bot 产能在 22-23。问题更像是：

- 前几回合海外争夺 + 高海外价格带来过高现金流。
- 市场收入分配继续回流预算池，形成连续滚雪球。
- Bot 没有有效跟进海外市场/军事争夺，导致玩家路线无竞争。
- 直接提交仍可能带入高效默认生产/政策，进一步放大优势。

建议：

- 重看海外胜利奖励：容量 `+8` 在早期库存 8 的情况下等于全量免费扩容，收益非常大。
- 考虑把胜利容量奖励改为随时代/外交等级增长，早期低于 8。
- Bot 必须能参与同一区域争夺，否则“竞争市场”实际只有玩家单边领奖。
- 如果保留高海外收益，需要增加对应风险，例如兵力消耗、外交摩擦、失败损失或区域封锁。

### P1：军事点、军队、事件加成三者语义混乱

UI 同时出现“军力”“军事点”“步兵/炮兵投入”，但它们的关系没有稳定表达：

- 地图卡“军事要塞 军力 2”在后端军队为 0 时仍显示 2。
- 军事面板的 `3/3`、`4/4` 与后端 `militaryPoints=2` 不一致。
- 海外争夺使用真实步兵/炮兵，不使用军事点；但玩家会被左侧军力误导，以为有兵可投。

建议：

- 地图卡区分“可用军事点”和“现有陆军”：例如 `军事点 2 / 陆军 0`。
- 海外争夺卡只引用现有陆军，不引用军力或军事点。
- 事件加成命名要明确是“本回合招募点数加成”还是“军队战力加成”。

## 已验证正常的行为

- 与中东建交后，中东卡开放销售与市场争夺控件。
- 没有陆军时，中东争夺控件展示“没有可投放陆军”，不能投兵。
- 欧洲可和平外销，但因未建交，市场争夺显示“需要先与该区域建交”。
- 争夺成功只扣胜者投入的步兵，收入按胜利价结算。
- 未争夺时，和平海外价格和海外容量仍能正常结算。
- 国内投放会按投放数量重算价格，不是固定参考价。
- 15 回合后能进入终局档案。

## 内容与体验问题

- “市民广场 / 市场预览”在当前主流程里仍显得存在感弱。如果它只是展示内需和价格基础，建议并入政府面板或贸易港，不要作为地图上同权重节点。
- “欧洲 开放市场”同时显示可出售、但市场争夺需要建交。逻辑是对的，但文案应区分“开放贸易”和“争夺准入”。
- 终局建议文案偏模板化，例如“抢节奏”“保市场权限”没有结合实际问题。本局真正的复盘应指出俄罗斯依赖海外出售收益，而不是泛泛说稳增长。
- 终局榜单显示俄罗斯总产能 8、其他国家 22+，但俄罗斯收入远高于其他国家。这个对玩家来说会显得评分体系被市场收入单点击穿，需要额外解释或调整。

## 修复优先级建议

1. 先修资源同源显示：政府财政、军事点、陆军数量必须与后端一致。
2. 阻断或显式化“未决策直接提交”，避免旧草稿/默认行为悄悄消费资源。
3. 重做海外区域卡的价格表达，把争夺成功价和保底价放在同一层级。
4. 调整海外胜利奖励与 Bot 参与逻辑，避免玩家单边滚雪球。
5. 再处理市民广场和政府政策的信息架构合并。
