## 当前玩法基线

### 回合结构

- 每局固定 15 回合。
- 每回合流程是 `国家决策 -> 市场出售 -> 财政结算`。
- 胜负核心仍是累计国家收入和终局排名。

### 经济与市场

- 当前实现以统一商品为主，不再让玩家管理煤炭、钢铁、粮食等多种具体商品。
- 生产链路是 `原材料 -> 统一商品`。
- 商品可以投放到国内市场或可进入的海外市场。
- 国内市场主要看需求、容量、价格和消费池支撑。
- 海外市场默认开放，主要看区域固定价、共享容量、市场争夺额外容量、地区封锁、政策和事件。
- 未售库存会跨回合结转，后续测试要继续确认结算页和下一回合入口是否表达清楚。

### 三资源池

- 民间购买力：支撑国内市场消费能力。
- 工厂预算：用于生产、原料、工厂增加、产业升级和临时工业调度。
- 政府财政：用于政府、研究、军事和海外通路相关行动。

### 工厂系统

- 工业路线包含手工业、机械化、蒸汽、电气化等阶段。
- `工厂增加`：把闲置工厂直接建成目标阶段工厂，本回合生效；不再有每类型扩建次数上限。
- 直接扩建成本按类型区分，当前为手工业 13、机械化 26、蒸汽 40、电气化 56。
- `产业升级`：把低级产能逐级转换为高级产能，可在本回合影响生产安排。
- 工厂只受国家总工厂上限、闲置名额、预算和科技前置限制，不再有单个行业工厂上限；例如英国可在总上限 16 内全部转为电气化工厂。

### 政府系统

- 行政力约束改革、常规政策和市场政策。
- 国内市场价格和软上限走 Phase 1 公式，不再保留 `市场补贴` / `价格管制` 这类国内市场政策入口。
- 政府市场政策当前只保留 `贸易促进`：提高海外容量，不增加海外价格。
- 政府财政现在只保留一个池子；政策、军事、研究和购买行政力都从同一政府财政扣除，不再显示或使用 `+8` 政策专项额度。

### 研究系统

- 研究设施提供研究进展。
- 首个发现某项科技的国家会进入突破判定。
- 后发国家在该科技已经被发现后，继续累积到原始阈值即可直接解锁。
- 研究失败、阈值变化和后续解锁应继续保持结算页反馈。

### 军事和海外

- 当前玩家可见主线应理解为陆军、舰队、地区封锁和海外市场通路。
- 军事动作当前不再提供 `海军演练` 海外扩容入口；海外容量主要由 `贸易促进`、市场竞争和其他长期效果体现。
- 建交不再作为海外出售或市场争夺前置；海外区域默认开放，真正限制来自容量和目标地区是否被封锁。
- 封锁操作以“点开海外地区并向该地区分配舰队”为入口，不再让玩家通过四大洋节点间接理解五个海外地区的封锁关系。
- 旧殖民、征服、掠夺链已经不应作为当前玩家可见主线；代码和测试中仍可能保留兼容字段或历史断言。
- 文档或 UI 中如果继续把当前版本描述为“征服/殖民地收益”主线，应视为过期口径。

### i18n 与语言

- 游戏中途不鼓励切换语言，避免活跃对局出现混合语言。
- 大部分核心游戏页已补齐中英文资源，但历史测试仍记录过 raw i18n key，需要真实页面复验。
- `/setting` 解释页按既定要求保留中文说明，不强制进入普通游戏页 i18n 路径。

## 下一轮需求修改前的判断

本轮修改入口是 `docs/修改索引.md`，核心修改依据是 `docs/本次核心修改文档.md`，执行状态表是 `docs/plans/2026-05-21-本轮修改计划与验证状态.md`，开发拆分是 `docs/plans/2026-05-21-市场工厂规则重构开发规划.md`。后续代码修改先按这些文档对齐，不再从旧阶段文档反推当前需求。

当前已经暂定的口径：

- 国内市场继续采用用户原始需求公式：消费池参与均衡价，供给短缺涨价，供给过剩降价；国内上限是软上限，不再硬拦截超额投放。
- 海外市场按新确认书改为固定价格；政策和市场竞争只增加海外容量，不增加海外价格。
- 建交删除后的海外准入已经确定：海外区域默认开放，价格也开放；市场出售只受库存、共享容量、竞争额外容量和地区封锁限制。
- 工厂总量按国家固定上限管理，新建工厂从闲置工厂中启用，每次 +1。
- 材料购买按国家设置单回合上限，购买只补本回合可用材料。
