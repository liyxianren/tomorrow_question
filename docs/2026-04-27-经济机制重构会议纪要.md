# 2026-04-27 经济机制重构会议纪要

> 会议主题：经济模拟游戏核心机制对齐与需求变更沟通会  
> 参与方：小宇老师、SCF  
> 文档定位：会后变更记录与后续开发裁决依据之一。  
> 重要结论：当前 Demo 的经济底层模型与客户预期存在根本差异，工业区与市场机制需要按供需模型重构。

## 1. 会议结论

本次会议确认：当前 Demo 中的经济机制不能作为最终验收目标。

主要原因：

- 当前价格机制偏随机或事件驱动，缺少供需关系基础。
- 当前产品体系包含多种具体商品，但客户预期核心模型应抽象为 `原材料` 与 `商品`。
- 当前市场机制没有体现“建厂既增加供给又创造需求”的宏观经济学逻辑。
- 简化后的确认文档不能被理解为删除原始复杂机制；客户认为原始复杂逻辑仍是核心需求。

## 2. 当前 Demo 与目标机制的差异

| 维度 | 当前 Demo / 简化实现 | 会后目标机制 |
| --- | --- | --- |
| 商品体系 | 多种具体商品，如钢铁、茶叶等 | 抽象为 `原材料` 与 `商品`，通过生产效率体现差异 |
| 价格形成 | 随机价格或事件影响价格 | 基于供给、需求、消费池资金形成动态价格 |
| 工厂作用 | 偏生产 / 扩产按钮逻辑 | 工厂既决定供给，也创造需求 |
| 市场策略 | 国内 / 海外销售入口已存在 | 需要通过供需差和价格变化驱动投放决策 |
| 验收状态 | 可作为 UI/流程 Demo | 不能作为核心经济机制验收版本 |

## 3. 已对齐的目标经济逻辑

### 3.1 供给端

供给由 `原材料数量 × 生产效率系数` 决定。

示例：

```text
机械厂效率 1:2
1 个原材料 -> 2 个商品
```

### 3.2 需求端

需求由 `产能结构` 决定。

关键点：

- 不同等级 / 类型的工厂不只产生商品，也会创造需求。
- 示例：机械厂产出 2 个商品，同时创造 3 个商品需求。
- 即使没有建厂，国家也存在基础需求，用于模拟人口基本消费。
- 示例：20 个空闲产能对应 20 个基础需求。

### 3.3 价格形成

均衡价格基准：

```text
均衡价格 = 上回合消费池资金 / 总需求
```

示例：

```text
720 资金 / 72 需求 = 10 均衡价格
```

价格波动：

- 供给小于需求：价格相对均衡价格上涨。
- 供给大于需求：价格相对均衡价格下跌。
- 示例：供给 64、需求 72，缺口为 8，缺口比例为 8 / 72 = 1/9，价格相对均衡价上涨 1/9。

> 具体公式、四舍五入、价格上下限、跨市场汇总方式，以《用户原始需求：核心机制》为当前原始依据；仍需确认四舍五入、价格上下限、跨市场汇总等实现边界。

### 3.4 市场投放策略

玩家需要决定商品投放到哪里：

- 本国市场
- 海外市场
- 殖民地 / 其他玩家市场（后续按原始文档确认）

策略目标：

- 若本国市场供过于求导致价格下跌，可将过剩产能转移到海外市场。
- 市场选择应影响利润、国家收入和后续预算池。

## 4. 后续开发范围

### 4.1 优先重构

1. `工业区 / 生产`
   - 合并具体商品概念，回到 `原材料 -> 商品` 的抽象模型。
   - 建立生产效率、工厂等级、需求创造之间的联动。
2. `市民广场 / 市场`
   - 重写供需关系与价格形成机制。
   - 支持市场投放策略对价格和收入的影响。

### 4.2 暂缓改动

以下模块暂时保持现状，后续再迭代：

- 议会厅
- 军事要塞
- 研究院 / 天赋树
- 大厅 / 房间 / AI 补位 / Ready 流程
- 基础 UI 框架与截图对齐页
