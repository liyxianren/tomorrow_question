# 文档入口

2. [修改索引.md](./修改索引.md)
   - 本轮大改的文档索引，说明需求依据、执行状态、验证流程和旧资料入口。

   - 本轮客户/策划确认过的核心修改依据，由最新确认书 DOCX 转换并同步。
4. [本轮玩法逻辑修改说明-客户版.md](./本轮玩法逻辑修改说明-客户版.md)
   - 面向客户说明本轮修改后的玩法、逻辑、数值和验证状态，不展开代码细节。
5. [plans/2026-05-21-本轮修改计划与验证状态.md](./plans/2026-05-21-本轮修改计划与验证状态.md)

## 读取规则

- 做新需求或大改动：先读 `当前状态总览.md`、`修改索引.md`、`本次核心修改文档.md`、本轮状态计划，再回到 `用户原始需求-核心机制.md` 判断方向是否跑偏。
- 做真实玩家测试或 bug 修复：先读 `playtests/QA_SESSION_PROTOCOL.md`，再看最近一次 playtest 记录。
- 查历史原因：看 `archive/旧文档总汇-2026-05-21.md`，不要把旧文档直接当成当前实现。
- 交给试玩用户：只发 `当前版本用户交接说明.md`，不要发阶段设计文档。

## 当前一句话口径

本项目当前是一个 10 回合历史策略游戏原型，核心发动机是 `原材料 -> 统一商品 -> 国内/海外销售 -> 三资源池回流`。政府、研究、工厂、军事、殖民和海外通路已经接入，后续改动要以 `5.27_overchange.md` 为最新玩法口径。
