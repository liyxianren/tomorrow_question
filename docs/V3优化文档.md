> 文档定位：V3 版本正式开发实施文档

> 前置依赖：V2 已完成（国际事件、意识形态、科技树、国家差异化）

## 1. 背景与目标

### 1.1 当前问题

军事系统分散在议会厅的政府策略中（征兵动员、远征军筹建、殖民特许），玩家感知不到军事力量的实际作用：
- 军事点影响海外区域准入，但前端没有展示
- 舰队影响海外承接力，但玩家看不到
- 没有"建交"概念，区域准入逻辑对玩家不可见
- 军事决策和政治策略混在一起，缺乏独立感

### 1.2 目标

将军事升级为**独立游戏模块**：
- 地图上新增第4个建筑"军事要塞"
- 专属军事决策面板（造舰、征兵、建交、远征）
- 海外区域准入状态可视化
- 从议会厅移除军事策略，各司其职

## 2. 产品设计

### 2.1 核心玩法循环

```
军事要塞（花政府财政预算）
├── 造舰队 → 增加海外市场承接力（能卖更多到海外）
├── 征兵 → 增加军事点数 → 满足受限区域准入条件
├── 建交 → 直接开放一个特定区域的贸易通道
└── 殖民远征 → 高成本，一次性获取大量海外优势
```

### 2.2 军事要塞内容分区

| 分区 | 内容 | 交互方式 | 预算来源 |
|------|------|---------|---------|
| 海外区域状态 | 5个区域的准入情况展示 | 只读展示 | — |
| 海军建设 | 建造舰队 | 点击确认（可多次） | 政府预算 |
| 陆军征募 | 征募步兵、训练炮兵 | 点击确认 | 政府预算 |
| 外交行动 | 与非洲/中东建交 | 点击确认（一次性） | 政府预算 |
| 军事远征 | 殖民远征 | 点击确认（每轮1次） | 政府预算 |

### 2.3 军事动作数值

| 动作 | 费用 | 效果 | 可重复 | 设计意图 |
|------|------|------|--------|---------|
| 建造舰队 | 8 政府预算 | 舰队+1（海外承接力+1） | 每轮最多3次 | 稳健提升海外出售能力 |
| 征募步兵 | 6 政府预算 | 军事点+1 | 每轮最多3次 | 低成本积累军事影响力 |
| 训练炮兵 | 12 政府预算 | 军事点+2 | 每轮1次 | 快速提升军事力量 |
| 与非洲建交 | **7** 政府预算 | 非洲区域永久开放 | 整局1次 | R3-4即可开辟新市场 |
| 与中东建交 | **7** 政府预算 | 中东区域永久开放 | 整局1次 | R3-4即可开辟新市场 |
| 殖民远征 | 20 政府预算 | 控制区域+1, 海外承接+2, 军事点+1 | 每轮1次 | 高投入一步到位 |

### 2.4 海外区域准入规则

| 区域 | 当前准入等级 | 准入条件 | 可售商品 |
|------|-----------|---------|---------|
| 欧洲 | 开放(open) | 无条件 | 煤炭、钢铁、粮食 |
| 美洲 | 开放(open) | 无条件 | 棉花、粮食、石油 |
| 非洲 | 特许(concession) | 军事点≥1 **或** 已建交 | 橡胶、棉花、矿产 |
| 中东 | 特许(concession) | 军事点≥1 **或** 已建交 | 石油、粮食、矿产 |
| 亚太 | 开放(open) | 无条件 | 茶叶、丝绸、煤炭 |

**海外市场承接力公式**：
```
海外承接力 = 海外承接加成(从策略效果) + 军事点 + 舰队数量
```

### 2.5 从议会厅移除的策略

| 移除的策略 | 替代方案 |
|-----------|---------|
| 征兵动员（8预算, 军事点+1） | 军事要塞：征募步兵（6预算） |
| 远征军筹建（20预算, 海外承接+2） | 军事要塞：殖民远征（20预算） |
| 殖民特许（34预算, 控制区域+2） | 军事要塞：建交 + 殖民远征 |

**议会厅保留**：贸易协定(6)、扩展航线(10)、内需刺激(10)、产业政策(12)、海军现代化(14)

新增函数 `_apply_military_plan`：

```python
def _apply_military_plan(
    player_state,
    military_plan: dict[str, Any],
    balance,
) -> int:
    remaining_budget = int(player_state.budget_pools.get("governmentFiscal", 0))
    spent = 0

    # 处理军事动作（造舰、征兵、炮兵、远征）
    action_counts: dict[str, int] = {}
    for selection in military_plan.get("militaryActions", []):
        action_id = str(selection.get("actionId") or "")
        action = balance.military_actions.military_actions.get(action_id)
        if action is None:
            continue
        count = action_counts.get(action_id, 0)
        if count >= action.max_per_round:
            continue
        if remaining_budget - spent < action.budget_pool_cost:
            continue
        spent += action.budget_pool_cost
        action_counts[action_id] = count + 1
        apply_effects(player_state, action.effects)

    # 处理外交建交
    for selection in military_plan.get("diplomacyActions", []):
        action_id = str(selection.get("actionId") or "")
        action = balance.military_actions.diplomacy_actions.get(action_id)
        if action is None:
            continue
        if action.target_region in player_state.established_diplomacy:
            continue  # 已建交，跳过
        if remaining_budget - spent < action.budget_pool_cost:
            continue
        spent += action.budget_pool_cost
        player_state.established_diplomacy.append(action.target_region)

    player_state.budget_pools["governmentFiscal"] = max(0, remaining_budget - spent)
    return spent
```

**注意**：军事花费和政府策略花费共享同一个预算池（`governmentFiscal`），需要在 `_apply_government_plan` 之后调用 `_apply_military_plan`，让它读取已扣除策略花费后的余额。

def _build_military_workspace(player, balance):
    return {
        "militaryPoints": player.military_points,
        "army": dict(player.army),
        "navy": dict(player.navy),
        "controlledRegions": player.controlled_regions_bonus,
        "establishedDiplomacy": list(player.established_diplomacy),
        "overseasCapacity": max(
            0,
            player.income_summary.get("overseasMarketCapacity", 0)
            + player.military_points
            + sum(int(v) for v in player.navy.values()),
        ),
        "regionAccessStatus": [
            {
                "regionId": region.region_id,
                "label": _get_region_label(region.region_id),
                "accessLevel": region.access_level.value,
                "isAccessible": _is_region_accessible_for_workspace(
                    region.access_level, player.military_points, region.region_id, player.established_diplomacy
                ),
                "isDiplomacyEstablished": region.region_id in player.established_diplomacy,
                "acceptedGoods": list(region.resource_limit.keys()),
