## 2. 本轮暂定规则口径

### 2.1 国内市场公式

当前没有玩家主动操作的“消费市场支出”，但消费池仍然参与市场价格计算。

```text
D_base = Σ(各生产方式产能数量 × 需求系数)
K = max(1, D_base + 国内承接能力修正)
C = 本回合进入市场阶段时的消费池资金
Q = 玩家投入国内市场的商品数量
P0 = C / K
```

其中需求系数来自用户原始需求：

| 生产方式 | 需求系数 |
| --- | ---: |
| 空闲产能 | 1 |
| 手工业生产 | 2 |
| 机械生产 | 3 |
| 蒸汽机生产 | 4 |
| 电气化生产 | 5 |

国内价格按原始需求的供需涨跌逻辑：

```text
当 Q < K：
短缺率 = (K - Q) / K
国内单价 = P0 × (1 + 短缺率)

当 Q = K：
国内单价 = P0

当 Q > K：
过剩率 = (Q - K) / K
国内单价 = max(0.1 × P0, P0 × (1 - 过剩率))
```

统一写法：

```text
P0 = 消费池 / K
国内单价 = clamp(P0 × (2 - Q / K), 0.1 × P0, 2 × P0)
```

暂定默认值：

- `P0` 最低按 1 处理，避免首回合正常投放低于 1；`最低价 = 0.1 × P0`，`最高价 = 2 × P0`，避免过剩时价格为负，也避免短缺时无限上涨。
- 国内市场上限是软上限：不再拦截 `Q > K`，只通过单价下降体现承接能力不足。
- 不额外存储“本回合过剩影响下回合价格”的惩罚字段。下回合价格通过收入分配进入消费池后自然变化。

国内销售收入：

```text
国内销售收入 = Q × 国内单价
```

### 2.2 海外市场公式

海外市场按本轮确认书执行：固定价格，不再受政府政策或市场竞争的价格加成影响。

```text
海外成交量 = min(投放量, 区域可出售上限)
海外销售收入 = 海外成交量 × 区域固定价格
```

暂定口径：

- 海外固定价格按区域配置。
- 如果需要体现国家差异，放在国家配置或区域价格表里，不通过政策临时加价。
- 政府政策、科技、事件、市场竞争只允许增加海外容量或出售上限，不增加海外单价。
- `overseasPriceBonusDelta`、`governmentOverseasPriceBonus`、市场竞争价格加成等玩家可见逻辑都要清理。

### 2.3 收入分配

用户原始需求明确初始分配为：

```text
消费 : 投资 : 财政 = 3 : 3 : 4
```

本轮按当前确认口径恢复 `3:3:4` 口径。

暂定结算：

```text
总销售收入 = 国内销售收入 + 海外销售收入
消费池增加 = 总销售收入 × 30%
工厂预算增加 = 总销售收入 × 30%
政府财政增加 = 总销售收入 × 40%
```

政策改变分配比例的历史能力如果仍保留，要继续走已有比例字段，而不是在市场公式里写死特殊分支。

### 2.4 工厂总量和闲置工厂

本轮确认书要求每个国家有固定工厂总量上限，且高级工厂来自低级工厂逐步升级。

暂定口径：

- 每国新增 `factoryTotalCap`。
- `idle` 表示未启用但存在的工厂容量。
- 初始 `idle = factoryTotalCap - 已启用工厂数量`。
- 新建工厂不是凭空增加总工厂数，而是从 `idle` 转为目标生产方式。
- 每次新建只 `+1` 目标生产方式，不再出现一次新建 `+2` 的隐性跳变。
- 每类型扩建次数上限和单个行业工厂上限已取消；扩建只受总工厂上限、闲置名额、预算和科技前置限制。
- 为平衡不限次数扩建，直接扩建成本按类型区分并小幅上调为手工业 13、机械化 26、蒸汽 40、电气化 56；兼容旧字段 `newFactoryCosts` 与当前 `expansionCosts` 保持一致。
- 升级只允许逐级转换：手工业 -> 机械 -> 蒸汽 -> 电气。
- 空闲产能按原始需求产生需求，默认需求系数为 `1`。
- 闲置需求系数、各国工厂上限放到 setting 供后续调参。

### 2.5 材料上限和一次性购买

本轮确认书要求材料上限国别不同，购买材料属于单回合行为。

暂定口径：

- 每国新增 `materialPurchaseCapPerTurn`。
- 玩家可以用工厂预算购买本回合可用材料。
- 购买量不能超过该国本回合材料购买上限。
- 购买材料不改变永久材料增长，也不改变长期材料上限。
- 默认数值先由开发估算，并在 setting 暴露。

示例方向：

| 国家 | 材料购买上限/回合 | 设计意图 |
| --- | ---: | --- |
| 英国 | 15 | 海外贸易强，但本土人口和资源承载有限 |
| 法国 | 16 | 均衡型 |
| 普鲁士 | 17 | 工业组织能力较强 |
| 奥地利 | 15 | 多民族帝国，稳定但效率一般 |
| 俄罗斯 | 20 | 原料充足，工业组织效率靠后期释放 |

### 2.6 封锁航线

封锁保留。原始需求中的封锁口径是：某玩家在海洋节点军力大于其他所有玩家时，可以封锁该节点，使其他玩家商品无法通过该节点流入区域市场。

暂定实现：

- 被封锁区域的海外可出售上限按 `0` 处理。
- UI 必须说明：
  - 哪个区域被封锁。
  - 被哪个关键海域或路线节点影响。
  - 当前影响是不能出售、容量为 0，还是竞争不可参与。
  - 如何解除：增加舰队、打破对方节点控制、等待事件或政策解除。

### 2.7 删除建交逻辑

确认书要求删除建交逻辑。当前已确认的替代口径是：海外区域默认开放，价格也开放；出售只受库存、共享海外容量、市场竞争额外容量和航线封锁限制。

开发拆分原则：

- 清理“建交加海外价格/容量”的误导文案和数据依赖。
- 海外出售、海外竞争、区域锁定原因都不能再依赖 `established_diplomacy`。
- 做全链路删除：配置、payload、后端校验、结算、前端按钮、提示、测试 fixture 和 i18n。

## 4. 开发阶段拆分

验收：

### 阶段 1：配置和 setting 基础

任务：

- 增加 `factoryTotalCap`。
- 增加 `materialPurchaseCapPerTurn`。
- 增加区域海外固定价格。
- 增加或整理 `minimumDomesticPrice`、`idleDemandCoefficient`。
- setting 页面能展示和编辑这些配置。

验收：

- 配置加载测试通过。
- setting 页面能说明这些参数的玩家含义。

### 阶段 2：海外价格和容量清理

任务：

- 从政府政策、研究、事件、能力中清理或替换 `overseasPriceBonusDelta`。
- 保留或迁移为 `overseasMarketCapacityDelta`，只影响海外容量。
- 市场竞争奖励从价格奖励改为容量奖励。
- 海外结算改为固定价格。
- 前端移除“海外价格 +N”的展示。

验收：

- 政府政策不再出现海外出售价格加成。
- 海外销售额严格等于成交量乘固定价格。
- 市场竞争只增加可出售上限。

### 阶段 3：国内软上限市场

任务：

- 后端移除 `domesticAllocation > domesticMarketCapacity` 的硬拦截。
- 前端移除国内投放 clamp。
- 市场结算按 `P = clamp(P0 × (2 - Q / K), 0.1×P0, 2×P0)` 计算国内单价。
- 预览区展示投放量、软上限、单价、过剩折价、预计收入。

验收：

- 玩家可以投放超过国内软上限。
- 超额投放成交但单价下降。
- 不出现负价格。
- 下回合价格只通过消费池变化间接变化。

### 阶段 4：工厂总量、闲置和材料购买

任务：

- 初始化各国 `idle`。
- 新建工厂从 `idle` 转目标生产方式，每次 +1。
- 升级只做相邻等级转换。
- 闲置工厂参与需求。
- 材料购买改为可选数量，并受国家上限约束。
- 工厂 UI 展示总上限、已启用、闲置、可购买材料量和成本。

验收：

- 工厂总数永远不超过国家上限。
- 新建工厂不会凭空新增总量。
- 闲置工厂会贡献少量需求。
- 材料购买不改变永久增长。

### 阶段 5：建交删除和海外准入

准入口径：

- 海外区域默认开放，区域固定价始终可见。
- 海外出售容量有限；市场争夺成功后额外 +2 可售容量。
- 航线封锁仍可让对应区域不可出售，容量视为 0。

任务：

- 删除 `diplomacyActions` 玩家入口。
- 清理 `established_diplomacy` 对海外市场准入和竞争的依赖。
- 更新旧测试 fixture。
- 不再引入“建交后才能出售/竞争”的替代门槛。

验收：

- 玩家流程中看不到建交按钮、建交提示、建交校验。
- 后端提交不再要求 established diplomacy。
- 海外区域锁定原因不再出现 `diplomacy_not_established`。
