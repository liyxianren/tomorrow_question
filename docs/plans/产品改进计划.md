## 二、修改计划

### Phase 1：核心循环修复（P0）

#### 1.1 产能可视化 + 原材料使用提示

**目标**：玩家一眼看到"产能限制了你能用多少原材料"

**改动**：
- 工厂面板顶部加产能条：`产能 8 / 原材料 25 → 最多用 8`
- 原材料剩余数字变色：有剩余时黄色警告
- 分配原材料时显示"受产能限制"tooltip

**文件**：
- `frontend/src/components/game/panels/factory/Phase1ProductionPanel.tsx`
- `frontend/src/components/game/panels/factory/Phase1ProductionPanel.css`

#### 1.2 市场信息简化

**目标**：玩家看到"你有 N 件商品，市场约需 M 件，预计售价 X"

**改动**：
- 市场面板顶部摘要改为：`商品 8 · 需求 8 · 预计售价 3`
- 去掉"均衡价""消费池"等术语
- 国内投放时实时显示：`投放 6 → 成交 6 · 单价 3 · 收入 18`
- 海外投放时显示：`亚太 · 售价 3 · 运力 5`

**文件**：
- `frontend/src/components/game/panels/Phase1MarketPanel.tsx`
- `frontend/src/components/game/panels/Phase1MarketPanel.css`

#### 1.3 消费池衰减叙事化

**目标**：用"国民消费"解释消费池减少，而非静默扣减

**改动**：
- 结算面板加一行：`国民消费 7 · 剩余消费池 25`
- 衰减公式不变（×0.7），但 UI 上用叙事包装
- 考虑改为：`消费池 = 上期余额 × 0.7 + 本期收入 × 0.5`

**文件**：
- `frontend/src/components/game/panels/GamePhasePanelContent.tsx`（SettlementWorkbench）
- `backend/app/modules/rules/settlement.py`（如改公式）

#### 1.4 第一回合增加可操作性

**目标**：让玩家第一回合有 4-5 个有意义的决策

**改动**：
- 初始科技点从 0 → 1（让研究面板有互动）
- 初始行政力从 2 → 3（让改革能做 1.5 个）
- 或：第一回合给 1 个免费改革（不消耗行政力）

**文件**：
- `backend/config/balance/countries.json`
- `backend/config/balance/global.json`

---

### Phase 2：引导与推荐（P1）

#### 2.1 改革入口突出

**目标**：让玩家看到"改革是本回合最重要的决策之一"

**改动**：
- 议会大厅加"推荐改革"标签（基于当前资源）
- 改革卡片加"首次推荐"高亮边框
- 改革列表按推荐度排序（而非固定路径顺序）

**文件**：
- `frontend/src/components/game/panels/GovernmentPanel.tsx`
- `frontend/src/components/game/panels/GovernmentPanel.css`

#### 2.2 国内动作推荐系统

**目标**：新玩家知道"第一回合推荐选什么"

**改动**：
- 国内动作卡片加"推荐"标签
- 推荐逻辑：
  - 第 1 回合：expand_workshop（扩产能）> market_fair（扩市场）
  - 第 2+ 回合：根据当前产能/需求比动态推荐
- 已选动作高亮，未选但推荐的加虚线边框

**文件**：
- `frontend/src/components/game/panels/DomesticPanel.tsx`
- `frontend/src/components/game/panels/DomesticPanel.css`

#### 2.3 国家差异化加大

**目标**：选国有明显的策略倾向

**改动**：

| 国家 | 特色 | 初始加成 |
|------|------|---------|
| 英国 | 海洋帝国 | +1 舰队，亚太外交，海外售价 ×1.2 |
| 法国 | 文化输出 | +2 消费池倍率，国内售价 +1 |
| 普鲁士 | 工业先发 | 1 机械化产能（2x 输出），+1 陆军 |
| 奥地利 | 多元治理 | +1 行政力，+1 研究设施 |
| 俄罗斯 | 资源丰富 | +5 原材料/回合，+2 产能 |

**文件**：
- `backend/config/balance/countries.json`

#### 2.4 海外市场引导

**目标**：有外交+舰队的玩家知道怎么卖海外

**改动**：
- 海外区域卡片加"可投放"高亮
- 显示：`运力 5 · 售价 3（×1.1）· 需要外交+舰队`
- 锁定区域显示具体解锁条件：`需要建交（5 财政）` 或 `需要军事点`

**文件**：
- `frontend/src/components/game/panels/Phase1MarketPanel.tsx`

#### 2.5 科技点来源说明

**目标**：科技点为 0 时，告诉玩家怎么获得

**改动**：
- 研究面板在科技点 = 0 时显示提示：
  - `科技点来源：研究院自动积累 · 改革"义务教育"（+1/回合）· 思潮里程碑`
- 科技点 > 0 时正常显示天赋树

**文件**：
- `frontend/src/components/game/panels/TalentTreePanel.tsx`

#### 2.6 研究/天赋树关系说明

**目标**：让玩家理解"链式研究"和"天赋树"是两回事

**改动**：
- 研究院面板分两个 tab：`科技链` 和 `天赋树`
- 科技链：自动推进（研究院积累进度），解锁新生产方式和商品
- 天赋树：手动解锁（花科技点），永久增益
- 两个 tab 的资源独立（科技链用研究设施，天赋树用科技点）

**文件**：
- `frontend/src/components/game/panels/TalentTreePanel.tsx`
- `backend/app/modules/rules/decision.py`（确认后端逻辑）

### Phase 3：深度优化（P2）

#### 3.1 国内面板精简

**改动**：
- 去掉"需求系数""效率""新建成本"等技术字段
- 每个动作只显示：名称、成本、效果标签、描述
- 效果标签用玩家语言：`+2 产能` 而非 `handicraftCapacityDelta: 2`

**文件**：
- `frontend/src/components/game/panels/DomesticPanel.tsx`

#### 3.2 军事第一回合可用性

**改动**：
- 招兵成本从 4 → 2（第一回合就能招 2 个）
- 或：第一回合给 1 个免费外交动作
- 外交从 5 → 3（更容易打开新市场）

**文件**：
- `backend/config/balance/military.json`

#### 3.3 大厅文案简化

**改动**：
- "最高统帅部收到的活跃作战房间电报" → "公开房间"
- "任务密钥" → "房间码"
- "前线响应状态" → "房间状态"
- 保留"集结入口""准备区"等氛围词

**文件**：
- `frontend/src/pages/RoomPage.tsx` 或相关组件

#### 3.4 demo 页同步更新

**改动**：
- 更新 DecisionCardDemoPage 使用 2.0 数据
- 或标记为"设计参考，非实际游戏数据"

**文件**：
- `frontend/src/pages/DecisionCardDemoPage.tsx`
- `frontend/src/features/game/demo/seed.ts`

## 四、验证标准

| 验证项 | 通过条件 |
|--------|---------|
| 第一回合可操作数 | ≥ 4 个有意义决策（工厂分配 + 国内动作 + 改革 + 军事或海外）|
| 新手理解测试 | 5 个新玩家能独立完成第一回合，不看攻略 |
| 国家差异感知 | 玩家能说出"选英国 vs 选法国有什么不同" |
| 市场决策效率 | 从进入市场到提交出售 ≤ 30 秒 |
| 消费池反馈 | 玩家能解释"为什么消费池变了" |
