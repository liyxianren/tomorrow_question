# 第一阶段：市场与生产机制

> 文档定位：第一阶段重构需求 / 设计文档。  
> 范围：只覆盖 `生产` 与 `市场` 两部分，以及它们对收入分配的最小闭环影响。  
> 原始依据：[用户原始需求：核心机制](./用户原始需求-核心机制.md)。  
> 会后依据：[2026-04-27 经济机制重构会议纪要](./2026-04-27-经济机制重构会议纪要.md)。  
> 目标模型参考：[目标经济模型：供给 - 需求 - 价格机制](./目标经济模型-供需价格机制.md)。

## 0. 一句话结论

第一阶段要把当前 Demo 从“多商品 + 随机 / 简化价格”的经营模型，改成用户原始需求中的宏观经济闭环：

```text
产能结构 -> 原材料转化为商品 -> 商品投放市场 -> 供需决定价格 -> 销售收入 -> 消费 / 投资 / 财政三池 -> 下一轮生产与市场
```

本阶段只解决两件事：

1. `生产怎么来`：原材料如何通过不同生产方式转化为商品。
2. `价格怎么来`：商品投放到市场后，供给、需求和消费池如何决定价格。

## 1. 第一阶段目标

### 1.1 产品目标

第一阶段完成后，玩家应能理解并体验到：

- 生产不是选择“钢铁、茶叶、煤炭”等具体商品，而是经营 `原材料 -> 商品` 的生产效率。
- 建厂 / 升级产能不仅提高供给，也改变本国需求结构。
- 市场价格不是随机数，而是供需关系的结果。
- 商品投放到本国市场、国际市场或殖民地，是围绕价格和承接能力做策略选择。
- 销售收入按制度比例进入 `消费池 / 投资池 / 财政池`，影响下一轮。

### 1.2 工程目标

第一阶段只重构以下系统：

- 生产方式与产能结构
- 原材料与商品抽象
- 本国市场供给、需求、均衡价格、供需价格波动
- 商品投放到本国 / 外部市场的最小结构
- 销售收入进入三资金池的最小闭环

第一阶段不处理完整政治、军事、科技树扩展，只保留它们作为后续阶段接口。

## 2. 当前 Demo 需要调整的方向

### 2.1 需要移除或弱化

当前 Demo 中以下内容不应继续作为核心经济模型：

- 多种具体商品作为核心经济对象：如钢铁、茶叶、煤炭、粮食等。
- 随机价格：如价格在某个区间内随机。
- 由国际事件直接决定商品价格的主逻辑。
- 只看库存与卖出按钮的简单销售模型。
- 将“工厂”只理解为增加产量，而不产生需求。

### 2.2 可以保留

以下内容可以作为 UI / 流程壳继续保留：

- 首页、大厅、房间、选国、Ready、AI 补位
- 主地图与可点击建筑入口
- 工业区作为生产入口
- 市民广场 / 贸易港作为市场入口
- 结算阶段外壳
- 议会厅、军事要塞、研究院入口暂时保留

### 2.3 本阶段重构重点

| 模块 | 当前问题 | 第一阶段目标 |
| --- | --- | --- |
| 工业区 | 多商品 / 生产按钮偏静态 | 改成产能结构、原材料、生产方式、商品产出 |
| 市民广场 / 市场 | 缺少供需定价 | 改成消费池、需求、供给、均衡价、价格波动 |
| 贸易港 | 只是销售入口 | 改成商品投放市场的决策台 |
| 结算 | 收入口径不够贴合原始机制 | 销售收入按 5:3:2 进入三资金池 |

## 3. 核心概念

### 3.1 原材料

`原材料` 是生产输入。

第一阶段不再把煤炭、钢铁、粮食、茶叶等具体物品作为主系统对象。它们可以在 UI 视觉或历史叙事中作为装饰，但不参与核心公式。

核心公式中只有：

```text
原材料 -> 商品
```

### 3.2 商品

`商品` 是统一的生产输出，也是市场销售对象。

商品不分种类。玩家需要决定的是：

- 生产多少商品
- 投放多少商品到本国市场
- 投放多少商品到外部市场
- 是否保留未售出商品

### 3.3 产能

`产能` 表示国家生产结构中的一个生产单位。

产能可以处于不同生产方式：

- 空闲产能
- 手工业生产
- 机械生产
- 蒸汽机生产
- 电气化生产

产能同时影响：

1. `供给能力`：能把多少原材料变成商品。
2. `需求创造`：会给本国市场创造多少商品需求。

### 3.4 消费池、投资池、财政池

销售收入进入三个资金池：

```text
消费 : 投资 : 财政 = 5 : 3 : 2
```

| 资金池 | 第一阶段用途 |
| --- | --- |
| 消费池 | 决定本国市场均衡价格基础 |
| 投资池 | 建设产能、升级生产方式 |
| 财政池 | 第一阶段暂不展开，只作为收入分配结果保留 |

## 4. 生产模型

### 4.1 生产方式表

第一阶段采用用户原始需求中的生产方式表：

| 生产方式 | 原材料 -> 商品 | 每单位产能创造需求 | 新建产能价格 | 科技门槛 | 第一阶段处理 |
| --- | --- | ---: | ---: | --- | --- |
| 空闲产能 | 不生产 | 1 | 待定 | 无 | 作为基础需求来源 |
| 手工业生产 | 1 -> 1 | 2 | 20 | 无 | 第一阶段默认可用 |
| 机械生产 | 1 -> 2 | 3 | 40 | 珍妮纺纱机 | 可先配置锁定/解锁状态 |
| 蒸汽机生产 | 1 -> 4 | 4 | 60 | 瓦特蒸汽机、车床 | 可先配置锁定/解锁状态 |
| 电气化生产 | 1 -> 8 | 5 | 80 | 发电、内燃机 | 可先配置锁定/解锁状态 |

### 4.2 产出计算

每种生产方式按自己的效率计算商品产出。

```text
某生产方式商品产出 = 投入该生产方式的原材料数量 × 该生产方式商品产出倍率
```

总商品产出：

```text
总商品产出 = Σ(各生产方式商品产出)
```

示例：

```text
手工业投入 10 原材料，效率 1:1 -> 产出 10 商品
机械生产投入 10 原材料，效率 1:2 -> 产出 20 商品
总产出 = 30 商品
```

### 4.3 原材料约束

玩家不能投入超过本回合可用原材料的数量。

```text
Σ(各生产方式投入原材料) <= 本回合可用原材料
```

若原材料不足：

- 不允许提交超过原材料上限的生产计划。
- UI 应提示“原材料不足”。

第一阶段暂不引入复杂原材料来源；可先使用每国固定初始原材料或配置表。

### 4.4 需求计算

需求由产能结构决定。

```text
本国总需求 = Σ(各生产方式产能数量 × 该生产方式需求系数)
```

需求系数：

| 生产方式 | 需求系数 |
| --- | ---: |
| 空闲产能 | 1 |
| 手工业生产 | 2 |
| 机械生产 | 3 |
| 蒸汽机生产 | 4 |
| 电气化生产 | 5 |

示例：

```text
空闲产能 20 -> 20 需求
手工业产能 10 -> 20 需求
机械产能 5 -> 15 需求
本国总需求 = 55
```

### 4.5 产能建设

玩家使用 `投资池` 建设新产能。

```text
新建成本 = 对应生产方式产能价格
```

第一阶段至少支持：

- 新建手工业产能
- 展示更高生产方式的价格和科技门槛
- 若科技未解锁，高级产能不可建

### 4.6 产能升级

用户原始需求：升级落后生产方式仅需投入半价。

第一阶段规则：

```text
升级成本 = 目标生产方式新建价格 × 50%
```

示例：

```text
手工业 -> 机械生产
机械生产新建价格 = 40
升级成本 = 20
```

待确认：

- 升级是否还需要扣除原生产方式残值。
- 是否允许跨级升级，如手工业直接到蒸汽机。
- 升级后的产能是否当回合立即生效。

第一阶段建议：

- 只允许升一级。
- 升级后的产能从下一回合生效。
- 当回合生产使用升级前产能结构，下一回合需求和供给能力再变化。

## 5. 市场模型

### 5.1 市场阶段输入

市场阶段需要以下输入：

- 本回合可销售商品数量
- 本国总需求
- 消费池资金
- 本国市场投放数量
- 外部市场投放数量
- 未投放 / 留存数量

约束：

```text
本国投放 + 外部投放 + 留存 <= 本回合可销售商品数量
```

### 5.2 本国市场供给

本国市场供给定义为：

```text
本国供给 = 本回合投入本国市场的商品数量
```

注意：

- 总产出不等于本国供给。
- 玩家可以把一部分商品投放到外部市场。
- 玩家也可以留存商品，是否允许留存及留存上限需配置。

### 5.3 本国市场需求

本国需求来自生产阶段确定的产能结构。

```text
本国需求 = Σ(产能数量 × 需求系数)
```

需求不是玩家在市场阶段手动填写的数值。

### 5.4 均衡价格

原始文档写法：

```text
均衡价格 = 本回合消费池资金 / 需求
```

会议口径中出现过“上回合消费池资金 / 总需求”。

第一阶段建议采用：

```text
均衡价格 = 阶段开始时消费池资金 / 本国需求
```

解释：

- “阶段开始时消费池”可以同时兼容“本回合开始状态”和“上轮结算后进入本回合的消费池”。
- 避免市场阶段销售收入立即改变本轮均衡价格，造成循环依赖。

待客户确认后，可将字段命名为：

```text
marketOpeningConsumptionPool
```

### 5.5 价格波动公式

当本国供给等于本国需求：

```text
本国价格 = 均衡价格
```

当本国供给小于本国需求：

```text
短缺率 = (本国需求 - 本国供给) / 本国需求
本国价格 = 均衡价格 × (1 + 短缺率)
```

当本国供给大于本国需求：

```text
过剩率 = (本国供给 - 本国需求) / 本国需求
本国价格 = 均衡价格 × (1 - 过剩率)
```

### 5.6 价格边界

为了避免过剩率超过 100% 导致负价格，第一阶段必须设置最低价格。

建议：

```text
最低价格 = 0
本国价格 = max(0, 计算价格)
```

如果需要更好体验，可以配置：

```text
最低价格 = 1
```

但这属于数值设计选择，需要客户确认。

第一阶段文档先按 `max(0, 计算价格)` 写入测试口径。

### 5.7 本国市场销售额

本国市场实际销售量建议取：

```text
本国实际成交量 = min(本国供给, 本国需求)
```

本国销售额：

```text
本国销售额 = 本国实际成交量 × 本国价格
```

待确认：

- 若供给大于需求，超出需求的商品是否滞销留存。
- 若供给小于需求，未满足需求是否只体现为涨价，还是影响思潮 / 独立度。

第一阶段建议：

- 本国超额投放部分不成交，进入未售出商品。
- 未售出商品是否保留到下回合，先配置化。

## 6. 外部市场最小模型

第一阶段重点是本国市场，但市场投放必须预留外部市场。

### 6.1 外部市场类型

原始需求中外部市场包括：

- 国际市场
- 殖民地
- 开放的其他玩家市场

第一阶段可以先抽象为：

```text
外部市场 = 若干个可投放市场列表
```

每个外部市场至少有：

- marketId
- marketName
- accessLevel：open / concession / colony
- demand
- consumptionPool 或 basePrice
- remainingCapacity
- owner / controller（可选）

### 6.2 第一阶段外部市场策略

第一阶段可实现最小版本：

- 本国市场用完整供需价格公式。
- 外部市场先使用配置化需求和价格。
- UI 支持把商品分配到本国与外部市场。
- 后续阶段再接入军事、殖民地、海军节点、封锁和独立度。

### 6.3 外部市场成交

建议第一阶段公式：

```text
外部市场成交量 = min(投放数量, 外部市场剩余需求或容量)
外部市场销售额 = 外部市场成交量 × 外部市场价格
```

待后续阶段扩展：

- 外部市场价格也按供需计算。
- 殖民地可掠夺原材料与产能。
- 其他玩家市场开放/关闭由政策控制。

## 7. 收入分配

### 7.1 总销售收入

```text
总销售收入 = 本国销售额 + 外部市场销售额总和
```

### 7.2 三池分配

第一阶段按照原始需求初始比例：

```text
消费 : 投资 : 财政 = 5 : 3 : 2
```

即：

```text
消费池增加 = 总销售收入 × 50%
投资池增加 = 总销售收入 × 30%
财政池增加 = 总销售收入 × 20%
```

### 7.3 分配时点

建议结算顺序：

1. 生产阶段得到商品产出。
2. 市场阶段分配商品投放。
3. 系统计算本国市场价格和销售额。
4. 系统计算外部市场销售额。
5. 汇总总销售收入。
6. 按分配比例进入三资金池。
7. 进入下一回合。

### 7.4 本阶段暂不处理

第一阶段不处理政策调整分配比例，只保留字段：

```text
allocationRatio = { consumption: 0.5, investment: 0.3, fiscal: 0.2 }
```

后续政治阶段再修改该比例。

## 8. 玩家操作流程

### 8.1 生产阶段玩家流程

玩家进入工业区后，应看到：

- 当前原材料
- 当前商品库存 / 本回合预计产出
- 消费池 / 投资池 / 财政池
- 各类产能数量
- 各类生产方式的效率、需求、建设价格、科技状态

玩家可以操作：

1. 分配原材料到不同生产方式。
2. 新建产能。
3. 升级产能。
4. 查看本回合预计商品产出。
5. 查看下一回合预计需求变化。

### 8.2 市场阶段玩家流程

玩家进入市场 / 贸易港后，应看到：

- 本回合可销售商品数量
- 本国需求
- 本国消费池
- 本国均衡价格
- 当前投放后预计价格
- 本国市场预计成交量与销售额
- 外部市场列表
- 未售出 / 留存数量

玩家可以操作：

1. 设置投放本国市场的商品数量。
2. 设置投放各外部市场的商品数量。
3. 查看价格与收入实时预估。
4. 提交市场分配方案。

## 9. UI 改造要求

### 9.1 工业区 UI

工业区不再展示多商品生产表，而应展示“产能结构”。

建议模块：

1. `资源概览`
   - 原材料
   - 商品库存
   - 投资池
2. `产能结构`
   - 空闲产能
   - 手工业
   - 机械生产
   - 蒸汽机生产
   - 电气化生产
3. `生产排程`
   - 各生产方式投入原材料
   - 预计商品产出
4. `建设与升级`
   - 新建产能
   - 半价升级
   - 科技锁定提示
5. `需求预览`
   - 当前需求
   - 建设 / 升级后下一回合需求

### 9.2 市场 UI

市场 UI 的核心是“投放后价格变化”。

建议模块：

1. `本国市场`
   - 消费池
   - 需求
   - 均衡价格
   - 投放数量
   - 短缺 / 过剩率
   - 预计价格
   - 预计销售额
2. `外部市场`
   - 市场名称
   - 进入权限
   - 需求 / 容量
   - 价格
   - 投放数量
   - 预计销售额
3. `收入分配预览`
   - 总销售收入
   - 消费池增加
   - 投资池增加
   - 财政池增加

## 10. 推荐数据结构

### 10.1 生产方式配置

```ts
type ProductionMode = {
  id: 'idle' | 'handicraft' | 'mechanized' | 'steam' | 'electric';
  label: string;
  inputRawMaterial: number;
  outputGoods: number;
  demandPerCapacity: number;
  buildCost: number | null;
  requiredTechIds: string[];
};
```

### 10.2 玩家经济状态

```ts
type PlayerEconomyState = {
  rawMaterials: number;
  goodsInventory: number;
  pools: {
    consumption: number;
    investment: number;
    fiscal: number;
  };
  allocationRatio: {
    consumption: number;
    investment: number;
    fiscal: number;
  };
  capacities: Record<ProductionMode['id'], number>;
};
```

### 10.3 生产提交

```ts
type ProductionSubmission = {
  rawMaterialAssignments: Record<ProductionMode['id'], number>;
  buildOrders: Array<{
    modeId: ProductionMode['id'];
    quantity: number;
  }>;
  upgradeOrders: Array<{
    fromModeId: ProductionMode['id'];
    toModeId: ProductionMode['id'];
    quantity: number;
  }>;
};
```

### 10.4 市场提交

```ts
type MarketSubmission = {
  domesticAllocation: number;
  externalAllocations: Array<{
    marketId: string;
    quantity: number;
  }>;
  retainedGoods: number;
};
```

### 10.5 市场计算结果

```ts
type MarketResolution = {
  domestic: {
    demand: number;
    supply: number;
    equilibriumPrice: number;
    shortageRate: number;
    surplusRate: number;
    finalPrice: number;
    soldQuantity: number;
    revenue: number;
    unsoldQuantity: number;
  };
  external: Array<{
    marketId: string;
    allocatedQuantity: number;
    soldQuantity: number;
    price: number;
    revenue: number;
  }>;
  totalRevenue: number;
  poolDelta: {
    consumption: number;
    investment: number;
    fiscal: number;
  };
};
```

## 11. 核心公式汇总

### 11.1 商品产出

```text
商品产出 = 原材料投入 × 生产方式产出倍率
```

### 11.2 本国需求

```text
本国需求 = Σ(各生产方式产能数量 × 需求系数)
```

### 11.3 均衡价格

```text
均衡价格 = 阶段开始时消费池资金 / 本国需求
```

### 11.4 价格

```text
如果供给 = 需求：
价格 = 均衡价格

如果供给 < 需求：
短缺率 = (需求 - 供给) / 需求
价格 = 均衡价格 × (1 + 短缺率)

如果供给 > 需求：
过剩率 = (供给 - 需求) / 需求
价格 = max(0, 均衡价格 × (1 - 过剩率))
```

### 11.5 销售额

```text
本国成交量 = min(本国供给, 本国需求)
本国销售额 = 本国成交量 × 本国价格
```

### 11.6 收入分配

```text
总收入 = 本国销售额 + 外部市场销售额
消费池增加 = 总收入 × 0.5
投资池增加 = 总收入 × 0.3
财政池增加 = 总收入 × 0.2
```

## 12. 示例推演

### 12.1 产能结构

假设某国当前产能：

| 生产方式 | 数量 | 需求系数 | 需求 |
| --- | ---: | ---: | ---: |
| 空闲产能 | 20 | 1 | 20 |
| 手工业 | 10 | 2 | 20 |
| 机械生产 | 10 | 3 | 30 |
| 合计 | 40 | - | 70 |

本国需求：

```text
20 + 20 + 30 = 70
```

### 12.2 生产

原材料分配：

| 生产方式 | 原材料投入 | 效率 | 商品产出 |
| --- | ---: | ---: | ---: |
| 手工业 | 10 | 1 -> 1 | 10 |
| 机械生产 | 20 | 1 -> 2 | 40 |
| 合计 | 30 | - | 50 |

总商品产出：

```text
50
```

### 12.3 市场

假设阶段开始消费池：

```text
消费池 = 700
需求 = 70
均衡价格 = 10
```

玩家向本国投放：

```text
本国供给 = 50
```

短缺：

```text
短缺率 = (70 - 50) / 70 = 28.57%
价格 = 10 × 1.2857 = 12.857
```

成交量：

```text
min(50, 70) = 50
```

销售额：

```text
50 × 12.857 = 642.85
```

三池增加：

```text
消费池 +321.43
投资池 +192.86
财政池 +128.57
```

> 实现时需要确认金额保留小数还是四舍五入为整数。

## 13. 验收标准

第一阶段完成后，至少应满足以下验收：

### 13.1 生产验收

- 不再以钢铁、茶叶、煤炭等具体商品作为核心生产对象。
- 系统存在统一的 `原材料` 与 `商品`。
- 不同生产方式有不同产出倍率。
- 不同生产方式有不同需求系数。
- 玩家可以使用投资池建设或升级产能。
- 升级成本按目标生产方式半价计算。

### 13.2 市场验收

- 本国市场价格不再随机生成。
- 本国需求由产能结构计算。
- 均衡价格由消费池和需求计算。
- 供给小于需求时价格上涨。
- 供给大于需求时价格下降。
- 玩家可以决定商品投放本国或外部市场。
- 销售收入进入消费 / 投资 / 财政三池。

### 13.3 UI 验收

- 工业区能清楚展示产能结构、生产方式、原材料、商品产出。
- 市场页能清楚展示供给、需求、均衡价格、短缺/过剩、预计收入。
- 玩家能理解“为什么涨价 / 为什么降价”。
- 页面不再让玩家误以为这是多商品库存买卖游戏。

### 13.4 测试验收

至少覆盖以下测试样例：

1. 机械生产 10 原材料，产出 20 商品。
2. 空闲 20 + 手工业 10 + 机械 10，需求为 70。
3. 消费池 700、需求 70，均衡价格为 10。
4. 供给 50、需求 70，价格上涨 28.57%。
5. 供给 90、需求 70，价格下降 28.57%。
6. 总收入 1000 时，消费 / 投资 / 财政增加为 500 / 300 / 200。
7. 过剩率超过 100% 时，价格不会低于最低价格。
