## 1. 世界地图

### 1.1 五个区域

| 区域 | 初始等级 | 所需航线 | 可殖民 | 特产资源 | 价格倍率 |
|------|---------|---------|--------|---------|---------|
| 欧洲 | 开放 | 无（陆路直达） | 否 | 煤5/钢4/粮3 | 0.9 |
| 美洲 | 特许 | 北大西洋 | 是 | 棉4/粮4/油2/钢3 | 1.0 |
| 非洲 | 特许 | 地中海 | 是 | 橡胶3/棉3/矿2 | 1.2 |

| 亚太 | 特许 | 印度洋 | 是 | 茶4/丝3/煤2 | 1.1 |

### 1.2 四个海洋节点 + 拓扑

```
北大西洋 ←→ 地中海 ←→ 印度洋 ←→ 太平洋
    ↕                    ↕           ↕
  美洲                 非洲        亚太
                       中东
```

**航线规则**：
- 到达"美洲"必须经过"北大西洋"
- 到达"非洲/中东"必须经过"地中海"
- 到达"亚太"必须经过"印度洋"
- "欧洲"不需要经过任何海洋节点（陆路直达，所有人均可到达）

### 1.3 市场等级含义

| 等级 | 谁能卖 | 谁能征服 | 成为殖民地后 |
|------|--------|---------|------------|
| 封闭 | 无人 | 无人 | — |
| 开放 | 所有人 | 可武力征服 → 变为己方特许 | — |
| 特许 | 仅建交者 | 可武力征服 → 变为己方殖民地 | — |
| 殖民地 | 仅所有者 | 可被他人武力夺取 | 所有者可掠夺资源 |

**等级变化路径**：
```
开放 ←→ 特许（外交建交 / 武力夺取）
特许 → 殖民地（武力征服）
殖民地 → 特许（叛乱独立）
```

## 2. 海军系统

### 2.1 舰队资产

- 舰队是永久性战略资产，不消耗
- 每回合决策阶段，玩家将舰队**部署**到海洋节点上
- 一个舰队只能部署到一个节点
- 部署上限：无硬限制，受拥有舰队总数约束
- 下回合可重新部署到不同节点（模拟舰队调动）

### 2.2 获取舰队

| 方式 | 成本 | 效果 |
|------|------|------|
| 造舰 `build_fleet` | 50$（从财政池扣） | navy.fleets + 1, militaryPoints + 1 |
| 海军演练 `naval_drill` | 6$ | militaryPoints + 1, overseasMarketCapacity + 1（不增加舰队） |

**设计意图**：造舰很贵（50$），但它是获取舰队的唯一永久途径。海军演练便宜且提供临时军事点和市场容量，但不造船。玩家需要权衡"砸钱造船控制航线"vs"省钱用演练维持存在感"。

### 2.3 初始舰队

| 国家 | 初始舰队 | 理由 |
|------|---------|------|
| 英国 | 1 | 海军传统 |
| 普鲁士 | 1 | 军事强国 |
| 法国 | 0 | — |
| 奥地利 | 0 | 内陆国 |
| 俄罗斯 | 0 | — |

### 2.4 部署与封锁结算

**时间线**（同一回合内）：

```
决策阶段：所有玩家同时提交 navalDeployment
         ↓
结算阶段：部署生效 → 封锁判定
         ↓
市场阶段：受到封锁影响的航线不可销售
```

**封锁判定规则**（按节点独立判定）：
1. 该节点海军最强的玩家记为 P
2. P 的海军数必须 **≥ 海洋控制阈值（2）**
3. P 的海军数必须 **严格大于** 该节点上所有其他玩家的海军数**之和**
4. 满足 1-3 → P 成为该节点的**封锁者**
5. 不满足 → 该节点**无封锁者**（自由通行）

**设计意图**：
- 条件 3 很苛刻——如果英国部署 3 舰队、法国部署 2 舰队，英国 3 > 2 但不大于法国之和（3 > 2 但 "严格大于所有其他之和" 这个条件，3 > 2，是的满足... wait，需要重新审视）

让我重新明确这个规则：

**修订版封锁规则**：
> P 的海军数必须严格大于该节点上**任意单个其他玩家**的海军数（即 P 是唯一的最高），且 P 的海军数 ≥ 阈值（2）。

理由：如果用"大于所有其他之和"，那几乎不可能封锁——需要绝对海军霸权。用"唯一最高 + 达阈值"更合理，同时也保留了"两个玩家都派 3 舰队则无人能封锁"的博弈张力。

**封锁示例**：

| 场景 | 北大西洋部署 | 封锁者 | 说明 |
|------|------------|--------|------|
| A | 英2 | 英国 | 唯一部署者 + 达阈值 → 封锁 |
| B | 英2 法1 | 英国 | 英2>法1 + 达阈值 → 封锁 |
| C | 英2 法2 | 无 | 平手 → 无人封锁 |
| D | 英1 法0 | 无 | 不足阈值(2) → 无人封锁 |
| E | 英3 法2 普1 | 英国 | 英最高(3) + 达阈值 → 封锁 |

### 2.5 封锁效果

- 被封锁的海洋节点：**除封锁者本人外**，任何玩家的商品不能通过该节点
- 通往该节点下游区域的所有航线被切断
- 封锁仅影响**商品流通**（市场销售），不影响外交建交、也不影响"路过"到达更远的节点（因为中间被卡住了）

**封锁对市场销售的影响**：
- 玩家在 market 阶段选择海外销售目标区域时
- 系统检查该区域的 requiredNodes 中是否有节点被**其他玩家**封锁
- 有 → 该区域不可销售
- 无 → 正常销售

## 3. 陆军与征服系统

### 3.1 陆军资产

- `army = {"infantry": N, "artillery": M}` 
- 陆军是永久性资产，用于征服殖民地
- 征服消耗步兵（从 army 中扣除）
- 炮兵是"质量兵力"——1 炮兵在征服中计为 2 个步兵的战斗力（体现火炮优势）

### 3.2 获取陆军

| 方式 | 成本 | 效果 |
|------|------|------|
| 征募步兵 | 10$ | infantry + 1, militaryPoints + 1 |
| 训练炮兵 | 8$（需要调，因为8$太便宜） | artillery + 1, militaryPoints + 2 |

**炮兵成本修正**：用户原文是"训练炮兵 8$→militaryPoints+2"。考虑到炮兵征服力 = 2 步兵，8$ 太便宜。建议改为 **16$**（= 2 步兵的成本）。

### 3.3 两种殖民地获取路径

#### 路径 A：和平殖民（现有，保留）

| 条件 | 值 |
|------|-----|
| 消耗 | 3 militaryPoints |
| 目标 | 仅无主殖民地（controller = None） |
| 前提 | 已解锁殖民（10$）+ 与该区域外交建交 |
| 限制 | 每回合最多 1 次 |
| 获得 | controller = 你, garrison = {"infantry": 1}（象征性驻军） |

#### 路径 B：武力征服（新增）

| 条件 | 值 |
|------|-----|
| 消耗 | 步兵兵力 + 炮兵（1 炮兵 = 2 战斗力） |
| 目标 | 无主区 OR 他人殖民地 |
| 前提 | 航线畅通（不被他人封锁） |
| 限制 | 无硬限制（受陆军存量约束） |
| 获得 | controller = 你, garrison = 所投入兵力 |

**武力征服兵力需求**：

| 目标类型 | 所需兵力 |
|---------|---------|
| 无主区域 | ≥ 1 兵力 |
| 他人殖民地 | ≥ 守军兵力 × 2 |

**征服优先级**：同一回合内，按玩家提交顺序处理。但建议改为"所有征服同时提交，冲突时兵多者得"——否则先手优势太大。

**征服冲突处理**（两人同时攻击同区域）：

```
如果目标无主：
  兵力多者获得 → 如果平手，都不获得（互相消耗）
  
如果目标是 A 的殖民地，B 和 C 同时攻击：
  B 需要 ≥ A守军×2，C 需要 ≥ A守军×2
  若都满足 → 兵力多者获得
  若平手 → 都不获得，A 保留
```

**战后处理**：
- 征服方消耗所有投入的兵力
- 胜利 → 征服方的投入兵力成为新驻军
- 败方/原主 → 如果原主被夺，原驻军清零（被消灭）

### 3.4 外交建交

- 与目标区域建交 → 写入 `established_diplomacy`
- 建交是**永久性**的，不消耗维护
- 建交成本：5$（从财政池扣）
- 和平殖民和特许市场销售都需要建交

### 3.5 海外市场销售规则

玩家可以向满足以下条件的区域销售：
1. 区域等级不是"封闭"
2. 如果是"特许/殖民地" → 玩家已与区域建交 或 玩家是区域的控制器
3. 如果是"开放" → 所有玩家均可
4. **航线畅通**（requiredNodes 中无节点被他人封锁）

## 4. 殖民地经济

### 4.1 殖民地收入

- 每个殖民地每回合提供 **5$** 国家收入（现有，保留）
- 殖民地收入在结算阶段自动计入 `national_income`

### 4.2 资源掠夺（殖民地专有）

**只有殖民地（非特许）的控制器可以掠夺资源**——这是成为殖民地 vs 特许的核心差异。

```
掠夺规则：
- 每回合，殖民地控制器可以选择从该殖民地掠夺一种资源
- 掠夺量 = min(玩家可掠夺上限, 该资源剩余量)
- 被掠夺的资源直接加入玩家的 raw_materials 池
- 掠夺会推高独立度（+2）
```

**资源再生**：每个区域有固定的资源上限（`resourceLimit`），被掠夺后不会自动恢复——这是"一次性矿脉"设计。这倒逼玩家不断寻找新殖民地。

| 区域 | 资源上限 |
|------|---------|
| 美洲 | 棉4/粮4/油2/钢3 |
| 非洲 | 橡胶3/棉3/矿2 |
| 中东 | 油4/粮2/矿2/钢2 |
| 亚太 | 茶4/丝3/煤2 |

**设计讨论**：一次性资源上限的设计会让殖民地在中后期"枯竭"。另一种方案是每回合恢复固定量。推荐**混合方案**——每个殖民地有一个"可再生基础量"和一个"矿脉上限"：

| 掠夺程度 | 恢复速度 | 效果 |
|---------|---------|------|
| 轻度掠夺（≤ 可再生量） | 完全恢复 | 可持续 |
| 重度掠夺（> 可再生量） | 不恢复 | 矿脉耗尽 |

但这个太复杂了。**简化版**：殖民地不提供自动资源，而是提供"掠夺行动"——每回合可以从殖民地掠夺一次，量有限，不恢复。这制造了"探索→掠夺→寻找新殖民地"的推进动力，符合 19 世纪殖民主义的气质。

## 5. 独立度系统

### 5.1 独立度变化公式

每回合结算阶段，对被控制的区域逐个计算：

```
delta = 市场失衡值 + 掠夺惩罚 - 驻军镇压

if delta != 0:
    independence = max(0, independence + delta)
    
if independence >= 阈值(10):
    叛乱！
```

### 5.2 市场失衡值

计算本回合该区域的供给与需求比率：

```
supply_total = sum(region.market_supply.values())
demand_total = sum(region.resource_limit.values())

ratio = supply_total / demand_total  (若 demand=0 则 ratio=1)

失衡值 = 
  0  （0.7 ≤ ratio ≤ 1.3：正常范围）
  +1 （1.3 < ratio ≤ 2.0 或 0.5 ≤ ratio < 0.7：轻微失衡）
  +2 （ratio > 2.0 或 ratio < 0.5：严重失衡）
```

**设计意图**：不论商品过剩还是短缺，经济不稳定都会推高独立度。殖民地居民需要稳定的经济环境。

### 5.3 掠夺惩罚

- 本回合掠夺了该殖民地的资源 → +2 独立度
- 不掠夺 → +0

### 5.4 驻军镇压

- 每 1 点 garrison（步兵或炮兵各算 1 点）→ -1 独立度
- 最小独立度 = 0

### 5.5 叛乱

叛乱发生时：
- 区域控制器变为 None
- 驻军清零（被消灭或撤退）
- 独立度归零
- 该区域等级从"殖民地"降为"特许"（但不是重新变为可建交状态——维持原外交关系）
- **对原主的外交关系保留**（原主仍可销售）

### 5.6 独立度信息可见性

- 每个区域的独立度对**所有玩家**可见（公开信息）
- 在 workspace 中暴露

## 6. 边界条件与边缘案例

| 场景 | 处理 |
|------|------|
| 玩家舰队 = 0，尝试部署 | 跳过（无法部署） |
| 两玩家同节点部署相同数量且都达阈值 | 平手 → 无人封锁 |
| 征服无主区，投入 0 兵力 | 拒绝（最少 1） |
| 征服他人殖民地，兵力不足 2×守军 | 拒绝，兵力不消耗 |
| 同一回合两玩家征服同一无主区 | 兵力多者得；平手则都不获得 |
| 同一回合夺取同一殖民地 | 兵力多者获得（都需满足 2×守军）；平手则原主保留 |
| 殖民地叛乱后立即被重新征服 | 允许（但刚叛乱独立度归零，短期不会再次叛乱） |
| 玩家被封锁后还能外交建交吗 | **不能**——无法到达的区域不能建交 |
| 舰队可以部署到没有接壤的节点吗 | **可以**——只要能到达节点本身。所有节点互相连通（见拓扑图），但到达区域需要经过特定节点 |
| 封锁者自己的商品是否能通过被封锁节点 | **能**——封锁只影响其他玩家 |
| 和平殖民和武力征服同一区域冲突 | 和平殖民先处理 → 如果成功则武力征服目标已改变（变为攻击殖民地的规则） |
| 资源掠夺后 region.resource_limit 归零 | 无法继续掠夺该资源 |
| 炮兵在征服中的战斗力 | 1 炮兵 = 2 战斗力（征服计算时；但不影响 garrison 镇压独立度——镇压只看数量） |

# 第二部分：数值平衡

## 7. 投入产出分析

### 7.1 建立一个殖民地的成本

| 路径 | 投入 | 回合数 |
|------|------|--------|
| 和平殖民 | 10$解锁 + 5$建交 + 3 militaryPoints（≈30$军备）≈ 45$ + 3回合 | 3-4 回合 |
| 武力征服（2 舰队 + 1 步兵） | 100$舰队 + 5$建交 + 10$步兵 = 115$ | 4-5 回合 |

### 7.2 殖民地回报

| 回报 | 每回合价值 |
|------|-----------|
| 固定收入 | 5$ |
| 资源掠夺（以煤为例） | 煤 1 单位 ≈ 1 商品 ≈ 均衡价格（取决于市场）≈ 5-10$ |
| 海外市场销售准入 | 取决于商品产量，约 10-20$/回合 |

**综合 ROI**：
- 和平殖民：45$ 投入，5$ 固定收入 + 市场销售 ≈ 15-25$/回合回报 → **2-3 回合回本**
- 武力征服：115$ 投入，同上回报 → **5-6 回合回本**

在 15 回合游戏中，两个路径都有正 ROI，但和平殖民回报更快。这符合设计意图——和平路径门槛低但只能拿无主区，武力路径门槛高但可以抢别人的。

### 7.3 封锁的经济价值

控制一个海洋节点 = 切断竞争对手的市场销售。

| 被切断区域 | 对手损失/回合（估算） |
|-----------|---------------------|
| 美洲 | 10-15$（海外销售收入） |
| 非洲 | 10-15$ |
| 中东 | 10-15$ |
| 亚太 | 10-15$ |

一架舰队 50$，控制一个节点只需 2 架 = 100$。如果能切断两个对手的航线，每回合让他们共损失 20-30$，**4-5 回合回本**。这是高回报但需要对手恰好依赖该航线。

**数值建议**：舰队成本 50$ 合理，但封锁阈值可以考虑调低到 1——让 1 架舰队就能封锁，降低门槛，让海军博弈更早发生。不过这会削弱"造第二艘船"的动力。

**最终决定**：保留阈值 2。这样玩家需要投入 100$（2 舰队）才能开始控制航线，与建立一个工厂的成本（20-80$）相当，是合理的中期投资。

### 7.4 独立度节奏

| 场景 | 每回合净变化 | 叛乱时间 |
|------|------------|---------|
| 无驻军 + 正常经济 + 不掠夺 | +1 | 10 回合 |
| 无驻军 + 失衡经济 + 掠夺 | +3~4 | 3 回合 |
| 1 驻军 + 正常经济 + 不掠夺 | 0 | 永不叛乱 |
| 1 驻军 + 失衡经济 + 掠夺 | +1~2 | 5-10 回合 |
| 2 驻军 | -1~-2 | 永不叛乱 |

**解读**：
- 1 驻军在正常运营下刚好维持稳定
- 如果同时掠夺 + 经济失衡 → 1 驻军不够，需要 2 驻军
- 玩家必须在"掠夺收益"和"驻军成本"之间权衡

**数值评价**：节奏合理。1 驻军（10$）维持一个殖民地稳定，相当于每回合 2$ 的维持成本（如果视驻军为沉没成本，摊到 15 回合 ≈ 0.7$/回合）。

| 文件 | 改动 |

| `decision.py` | 在 `_apply_military_plan` 中插入（殖民之后） |

**征服逻辑详细流程**：
```
1. 收集所有玩家本回合的 conquestActions
2. 按区域分组 → 检测冲突
3. 无冲突：逐个执行
4. 有冲突（多人攻击同一区域）：
   - 计算每个攻击者的有效兵力（步兵 + 炮兵×2）
   - 兵力最高者获得；平手 → 无人获得
5. 胜利者：消耗兵力 → controller = 自己 → garrison = 投入兵力
6. 原主（若有）：garrison 清零
```

**验证**：新单元测试（征服无主区、2×守军夺取、兵力不足拒绝、炮兵战斗力=2、两人冲突兵多者得、平手无人得）

### Task 4：规则层 — 独立度变化 + 叛乱

| 文件 | 改动 |
|------|------|
| `settlement.py` | 新增 `_apply_independence_progression(snapshot, balance)` — 市场失衡计算 → 掠夺惩罚 → 驻军镇压 → 叛乱判定 |
| `settlement.py` | 在收入分配之后、Phase 3 研究进展之前插入 |
| `models.py` | 叛乱后：region.controller → None, garrison → {}, independence → 0, access_level → CONCESSION |

**独立度计算**：
```
for each controlled region:
    ratio = supply / demand
    delta = 0
    if ratio > 2.0 or ratio < 0.5: delta += 2
    elif ratio > 1.3 or ratio < 0.7: delta += 1
    if was_looted_this_turn: delta += 2
    delta -= sum(garrison.values())
    independence = max(0, independence + delta)
    if independence >= 10: revolt!
```

**验证**：新单元测试（正常经济独立度稳定、失衡推高、掠夺加速、驻军镇压、超阈值叛乱、叛乱后状态重置）

```

```
