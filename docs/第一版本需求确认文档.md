# 第一版本需求确认文档

## 1. 文档目的

本文件用于确认项目首版的产品边界、玩家体验目标和正式规则主线。  
首版规则统一采用《交付确认文档 2.0》的国家收入制。

## 2. 首版产品目标

首版要交付的是一个可完整跑通的多人网页策略游戏：

- `5` 人固定房间
- `15` 回合完整对局
- 同步提交
- 统一结算
- 支持刷新恢复
- 支持大厅、房间、对局、结算的完整链路

## 3. 首版核心体验

玩家在每回合都要面对三层国家经营压力：

1. `工厂`
   - 生产够不够
   - 扩产值不值
   - 工业路线要不要升级
2. `国内消费市场`
   - 国内市场有没有足够的承接能力
   - 是否要优先扶持内需
3. `政府财政`
   - 科技点和军事点怎么购买
   - 科技、军事和国家政策怎么取舍

玩家的核心体验，不是操控一个万能钱包，而是经营一个不断变化的国家收入结构。

## 4. 正式规则基线

### 4.1 回合结构

首版正式回合结构固定为：

`国家决策 -> 市场出售 -> 财政结算`

### 4.2 国家收入

国家收入定义为：

`本回合国内销售额 + 本回合海外销售额`

### 4.3 国家收入分配

财政结算后，国家收入按当前比例自动分配到三个独立预算池：

1. `国内消费市场`
2. `工厂`
3. `政府财政`

初始比例为：

`3 : 3 : 4`

### 4.4 第一阶段决策类型

首版第一阶段固定包含三类主动决策：

1. `工厂决策`
2. `国民消费决策`
3. `政府财政决策`

并且明确：

- 国民消费决策消耗 `国内消费市场`
- 政府财政决策消耗 `政府财政`
- 两者都可能影响国内市场，但不是同一件事

### 4.5 第二阶段市场出售

市场出售阶段由玩家主动决定：

- 卖向
- 卖量
- 是否承担压仓库风险

### 4.6 政府策略影响国家结构

政府财政动作可以改写收入分配比例。  
例如海外扩张可让国家从 `3 : 3 : 4` 变成 `2.8 : 3 : 4.2`，让结构更偏财政与海外方向。

### 4.7 胜负条件

主胜负条件固定为：

**15 回合累计国家收入最高者获胜**

## 5. 首版必须交付的功能

- 大厅页
- 房间创建 / 加入 / 选国 / Ready
- 对局页三阶段主循环
- 实时同步与日志
- 阶段结果摘要
- 最终结算页
- 刷新恢复

## 6. 首版暂不纳入范围

- 账号体系
- 匹配
- 观战
- 历史战绩
- 税收、人口、劳动力等更深层系统
- 多实例扩容

## 7. 实现导向

后续技术方案、数据结构、接口设计、前端产品化和测试验收都必须围绕以下事实展开：

- 国家收入是唯一结算口径
- 三个预算池是独立池
- 第一阶段固定三类主动决策
- 市场出售是玩家主动阶段
- 财政结算负责汇总收入并重新分配预算
