# 2.0 阶段 2 实施计划

> 文档定位：阶段 2 可执行实施计划  
> 覆盖范围：前端运行时与工作台迁移  
> 前置条件：阶段 1 共享契约与后端经济内核已冻结  
> 不覆盖范围：恢复链、实时链、Bot 链、超时补交链、浏览器整链收口

## 1. 计划目标

本计划只服务一个目的：

**在阶段 1 契约冻结后，把当前“新三阶段外壳 + Treasury 兼容内容”的前端主链路，迁移成完整 2.0 的国家收入制工作台。**

本计划是 [开发任务清单](../../开发任务清单.md) 中阶段 2 的唯一实施依据，也是阶段 3 接管恢复、实时、Bot 与测试链之前的前端主链路冻结文档。

## 2. 阶段 2 完成定义

阶段 2 完成后，前端必须满足以下条件：

1. `GamePage` 主流程只保留 `decision / market / settlement`
2. 主类型、草稿状态和页面文案不再保留 Treasury 主语义
3. 国家决策台、市场出售台、财政结算台都能按 2.0 快照正确渲染
4. 排行和结算页统一按 `cumulativeNationalIncome`
5. 阶段 3 可以在不重写主页面的前提下，直接接入恢复、Socket、Bot 和自动补交链

## 3. 阶段范围与边界

### 3.1 本阶段覆盖

- 前端共享类型清理
- 草稿结构与页面状态机迁移
- 三块主工作台迁移
- 顶部资源带、阶段标签、任务条、命令坞文案迁移
- 排行与结算页的 2.0 口径切换
- 与上述改动直接相关的最小前端组件测试
- 页面草稿、工作台 union、legacy 四阶段 UI 类型和文案清退
- 凡是页面草稿、workspace union、view model、页面状态机和 UI 文案清理，统一归阶段 2

### 3.2 本阶段不覆盖

- `useGameRuntime` 的恢复链与 Socket 事件消费重构
- `connectSocket`、`runtime/model.ts` 的实时语义清理
- Bot、超时补交、浏览器整链回归
- 新工作台视觉重设计
- 后端共享契约定义与 [frontend/src/types/domain.ts](../../../frontend/src/types/domain.ts) 的基础 schema 裁决
- 任何直接映射后端 payload / snapshot 的共享 schema 裁决

## 4. 目标公共接口与前端边界

阶段 2 只消费已经冻结的 2.0 契约：

- `GamePhase = decision | market | settlement`
- `GameSnapshotPayload` / `PlayerStatePayload` 已包含：
  - `domesticSalesRevenue`
  - `overseasSalesRevenue`
  - `nationalIncome`
  - `cumulativeNationalIncome`
  - `incomeAllocationRatio`
  - `budgetPools.domesticMarket`
  - `budgetPools.factory`
  - `budgetPools.governmentFiscal`
  - `techPoints`
  - `militaryPoints`
- 提交契约只接受：
  - `DecisionSubmission`
  - `MarketSubmission`
- `settlement` 为只读阶段，无玩家提交

## 5. 任务群 A：类型与草稿层清理

### 5.1 目标

先清理前端主类型和草稿状态，把 legacy 四阶段和 Treasury 兼容字段从页面主链路移除。

### 5.2 文件边界

- [frontend/src/types/domain.ts](../../../frontend/src/types/domain.ts)
- [frontend/src/features/game/forms.ts](../../../frontend/src/features/game/forms.ts)
- [frontend/src/features/game/labels.ts](../../../frontend/src/features/game/labels.ts)

### 5.3 实施步骤

- 清退 `GamePhase` 中的 `production / military / politics`
- 清退 `fundPools`、`treasuryBalance`、`cumulativeTreasuryIncome`、`pendingSettlementIncome`
- 清退 `domesticStrength / industryStrength / statecraftStrength`
- 把草稿结构收敛为：
  - `DecisionPhaseDraft`
  - `MarketPhaseDraft`
  - `SettlementPhaseDraft`
- `DecisionPhaseDraft` 固定承载：
  - 工厂决策输入
  - 国民消费决策输入
  - 政府财政决策输入
- 本任务群才开始处理页面草稿、工作台 union、legacy 四阶段 UI 类型和文案清退，不回头承担后端共享契约裁决

### 5.4 验收标准

- `domain.ts` 不再把 legacy 四阶段当正式类型
- `forms.ts` 不再生成四阶段 draft
- 标签层不再输出旧四阶段中文名

## 6. 任务群 B：页面状态机与摘要层

### 6.1 目标

让页面控制层和公共摘要层只按 2.0 的三阶段、三池、国家收入制运转。

### 6.2 文件边界

- [frontend/src/features/game/flow/useGamePageController.ts](../../../frontend/src/features/game/flow/useGamePageController.ts)
- [frontend/src/pages/GamePage.tsx](../../../frontend/src/pages/GamePage.tsx)
- [frontend/src/components/game/status/GameSituationSummary.tsx](../../../frontend/src/components/game/status/GameSituationSummary.tsx)

### 6.3 实施步骤

- `useGamePageController` 只保留三阶段 draft/reset
- `GamePage` 的阶段说明、任务条、命令坞、主舞台文案全部切到：
  - 国家收入
  - 国内消费市场
  - 工厂
  - 政府财政
  - 科技点
  - 军事点
  - 当前收入分配比例
- `GameSituationSummary` 继续负责回合/阶段/连接/计时展示，但不再承担 Treasury 风格说明

### 6.4 验收标准

- 主页面不会再显示“国库 / 累计回流 / 国家强度”
- `settlement` 继续保持只读，不出现提交流程

## 7. 任务群 C：三块工作台

### 7.1 目标

把 `GamePhasePanelContent` 收敛成完整 2.0 三块主工作台。

### 7.2 文件边界

- [frontend/src/components/game/panels/GamePhasePanelContent.tsx](../../../frontend/src/components/game/panels/GamePhasePanelContent.tsx)

### 7.3 实施步骤

- `国家决策台`
  - 工厂决策区
  - 国民消费决策区
  - 政府财政决策区
  - 同屏展示三池余额、点数余额、比例变化预告
- `市场出售台`
  - 商品库存
  - 国内 / 海外卖向
  - 卖量
  - 压仓库风险
  - 国家收入构成预估
- `财政结算台`
  - 本回合国内销售额
  - 本回合海外销售额
  - 本回合国家收入
  - 三池重新分配结果
  - 当前比例变化
  - 累计国家收入

### 7.4 验收标准

- 玩家能从三个面板直接理解三阶段主循环
- 工作台不再使用 Treasury 成本和强度值作为主解释口径

## 8. 任务群 D：排行、结算页与最小前端测试

### 8.1 目标

完成阶段 2 范围内的页面收口，但不提前吞并阶段 3 的恢复与整链测试。

### 8.2 文件边界

- `frontend/src/pages/SettlementPage.tsx`
- `frontend/src/test/*`

### 8.3 实施步骤

- 排行主指标统一改为 `cumulativeNationalIncome`
- 结算页统一展示：
  - 各国 `15` 回合累计国家收入
  - 关键政府策略与结构变化复盘
- 只补足阶段 2 所需的最小前端验证：
  - 类型测试
  - 页面控制器测试
  - 三阶段工作台渲染测试

### 8.4 验收标准

- 结算页与排行页不再使用 Treasury 主语义
- 阶段 2 改动有足够的前端单测保障
- 恢复、实时、Bot、E2E 仍留给阶段 3 统一收口

## 9. 本阶段禁止回退的旧模型

本阶段执行过程中，不允许再把以下概念当作正式前端实现方向：

- `production / military / politics`
- `fundPools`
- `treasuryBalance / cumulativeTreasuryIncome / pendingSettlementIncome`
- `domesticStrength / industryStrength / statecraftStrength`
- 旧四阶段中文文案与过渡期提示

## 10. 阶段 2 完成后的移交

阶段 2 完成后，后续移交边界固定为：

- 主页面、工作台、结算页已冻结为 2.0 结构
- 阶段 3 只接管：
  - 恢复
  - 实时
  - Bot
  - timeout
  - 整链测试
- 阶段 3 的唯一实施依据固定为 [2026-04-06-2.0-phase3-runtime-recovery-tests.md](./2026-04-06-2.0-phase3-runtime-recovery-tests.md)
