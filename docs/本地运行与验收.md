# 本地运行与验收

## 1. 本地运行目标

本地联调的目的，是验证国家收入制下的完整玩家链路：

- 大厅
- 房间
- 选国 / Ready
- `15` 回合三阶段对局
- 结算
- 刷新恢复

## 2. 启动方式

建议直接使用仓库脚本。

### 安装依赖

- 后端依赖：`backend/requirements.txt`
- 前端依赖：`frontend/package.json`

### 启动本地联调环境

```powershell
./scripts/start-local.ps1
```

默认检查地址：

- 前端：[http://127.0.0.1:5173](http://127.0.0.1:5173)
- 后端健康检查：[http://127.0.0.1:5000/healthz](http://127.0.0.1:5000/healthz)

## 3. 手动验收流程

### 房间链路

1. 进入大厅
2. 创建房间
3. 让 `5` 名玩家加入
4. 固定选国
5. 所有人 Ready

### 对局链路

1. 进入 `国家决策`
2. 确认能看到三个预算池：
   - 国内消费市场
   - 工厂
   - 政府财政
3. 完成三类主动决策：
   - 工厂决策
   - 国民消费决策
   - 政府财政决策
4. 进入 `市场出售`
5. 提交卖向与卖量
6. 查看是否正确产生国内销售额、海外销售额与国家收入
7. 进入 `财政结算`
8. 查看三池是否按当前比例重新分配

### 恢复链路

1. 在任意阶段刷新页面
2. 确认玩家能回到正确回合与正确阶段
3. 确认三池、点数、库存与当前比例都能恢复

### 超时与自动补交链路

1. 在任意可提交阶段等待至少一名玩家超时
2. 确认系统能自动补交并推进到下一阶段
3. 如房间内包含 Bot，确认 Bot 可以持续自动生成提交
4. 确认自动补交后，排行榜与结算不会丢失

### 最终结算

1. 跑满 `15` 回合
2. 确认最终排名按累计国家收入排序
3. 确认结算页能展示各国表现摘要

## 4. 辅助脚本

### 快速检查

```powershell
./scripts/check-local.ps1 -Mode smoke
./scripts/run-browser-tests.ps1 -Mode smoke
```

### 停止本地环境

```powershell
./scripts/stop-local.ps1
```

### 完整测试套件

```powershell
./scripts/run-test-suite.ps1
```

### 正式签收门槛

```powershell
./scripts/run-tests.ps1
./scripts/run-browser-tests.ps1 -Mode full
./scripts/check-local.ps1 -Mode full
./scripts/run-test-suite.ps1
```

## 5. 验收结论标准

本地验收通过至少要满足：

- 三阶段可以连续推进
- 三池可以正确展示和扣减
- 国家收入可以正确生成和结算
- 政府策略可以改变收入结构
- 页面刷新后状态不丢失
- 刷新后 `round / phase / submissionStatus / latestSettlement` 与恢复接口一致
- 超时补交后恢复接口可见 `autoSubmittedPlayerIds`
- `15` 回合后能正常结算
- 最终排名与 `/final-result` 返回的累计国家收入排序一致
- `run-browser-tests.ps1 -Mode full` 全绿
- `check-local.ps1 -Mode full` 全绿
- `run-test-suite.ps1` 一次通过
