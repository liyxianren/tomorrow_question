# 前端产品化改造设计

## 1. 设计目标

前端产品化改造的目标，是把新版国家收入制变成玩家能看懂、能操作、能复盘的对局界面。

对局主舞台必须围绕三件事展开：

1. 现在处于哪个阶段
2. 三个预算池分别还能做什么
3. 本回合国家收入与结构变化是什么

## 2. 页面结构

首版路由结构保持：

- `Home`
- `Lobby`
- `Room`
- `Game`
- `Settlement`

其中改造重点集中在 `Game` 与 `Settlement`。

## 3. Game 页主结构

### 顶部信息带

固定展示：

- 当前回合 / 总回合
- 当前阶段
- 当前国家
- 当前收入分配比例
- 本回合国家收入摘要

### 核心资源带

固定展示三池与点数：

- `国内消费市场`
- `工厂`
- `政府财政`
- `科技点`
- `军事点`

设计要求：

- 三池必须并列展示
- 不能再做成一个主钱包加若干附属标签
- 比例变化必须有可视反馈

### 阶段工作台

根据阶段切换三类主面板：

1. `国家决策台`
2. `市场出售台`
3. `财政结算台`

## 4. 国家决策台

国家决策台必须同时容纳三类主动决策：

### 工厂决策区

- 安排生产
- 支付工业用工成本
- 开办新厂
- 扩产
- 升级工业路线

### 国民消费决策区

- 选择扶持国内消费动作
- 预览下一轮国内市场能力变化
- 预览国内市场收购价格变化

### 政府财政决策区

- 购买科技点
- 购买军事点
- 选择科技策略
- 选择军事策略
- 选择扶持国内市场政策

设计要求：

- 三类动作的预算来源要写清楚
- `国民消费决策` 和 `政府扶持内需政策` 不能做成同一个交互块
- 提交前必须展示本回合将消耗哪些预算池

## 5. 市场出售台

市场出售台负责把“生产结果”转成“国家收入”。

必须支持：

- 商品列表
- 当前库存
- 国内 / 海外卖向分配
- 每类商品卖量输入
- 卖不出去的压仓库提示

页面必须能让玩家直接理解：

- 国内卖了多少
- 海外卖了多少
- 哪些商品被压仓库

## 6. 财政结算台

财政结算台为只读面板，重点是结算表达。

必须展示：

- 本回合国内销售额
- 本回合海外销售额
- 本回合国家收入
- 三池各自分到多少
- 当前比例是否发生变化
- 累计国家收入

设计要求：

- 结算页不是数据表堆砌
- 要让玩家一眼看懂“我这回合赚了多少钱，钱去了哪里”

## 7. 文案与反馈原则

- 全部前台术语统一用中文
- 不再出现历史英文三池命名
- 不再出现历史阶段名称
- 阶段说明与按钮说明都要强调预算来源与作用对象

## 8. 后续扩展兼容

当前前端结构要为后续加入下列系统留口子：

- 人口 / 劳动力
- 税收
- 更复杂的商品树
- 更复杂的海外市场

扩展方式应优先通过面板增量扩展，不破坏“三阶段 + 三池”主布局。
