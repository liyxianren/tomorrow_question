# Frontend Productization Workstream

## 定位

这组文档用于把前端工作从“技术原型修补”中拆出来，转成独立的产品化交付工作流。

它们的目标不是解释后端规则，而是回答以下问题：

- 前端现在应该是什么产品形态
- 首页、大厅、房间、对局、结算分别承担什么职责
- 前端为什么不能继续按原型页面补丁推进
- 后续 `subagent` 应该按什么任务树并行开发前端

## 文档入口

前端专项文档固定为：

- `docs/前端产品化改造设计.md`
- `docs/前端产品化开发任务清单.md`

## 何时优先读这组文档

当用户在问以下问题时，应优先进入这组文档，而不是直接回到总任务清单：

- 首页、大厅、房间、对局页面怎么重构
- 当前前端为什么不具备交付品质
- 前端 UI / IA / 文案 / 体验应该怎么改
- 前端应该怎么拆分给多个 `subagent`
- 前端专项和主项目开发任务之间是什么关系

## 与主项目文档的关系

这组文档与通用项目文档的关系如下：

- 产品边界仍以上位文档为准：
  - `docs/最终交付确认文档.md`
- 系统边界仍以上位文档为准：
  - `docs/技术方案.md`
  - `docs/接口与事件契约设计.md`
- 但前端实现路径、页面职责、设计系统和前端测试路径，以这组专项文档为直接执行依据

## 回答策略

- 如果用户问“前端现在应该看哪份文档”，先指向 `前端产品化改造设计.md`
- 如果用户问“前端怎么开始开发”，再指向 `前端产品化开发任务清单.md`
- 如果用户问“为什么前端不直接看总任务清单”，回答：因为前端已拆成专项工作流，需要与通用开发任务树解耦，避免继续把产品体验问题混进系统实现任务里
