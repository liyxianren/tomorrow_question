# Project Brief

## 项目简介

`tomorrow_question` 是一个 19 世纪工业化列强竞争题材的网页回合制策略游戏 MVP。

核心交付目标不是规则展示，而是一个可以完整跑完一局的可玩网站游戏，重点是：

- `5 人固定房间制`
- `15 回合`
- `同步提交，统一结算`
- `生产、科技、市场、军事、政治` 五大系统
- 文字/节点版地图逻辑
- 排行榜、日志、最终结算

当前如果推进前端交付，不应直接在通用任务树里硬修页面，而应进入以下专项文档：

- `docs/前端产品化改造设计.md`
- `docs/前端产品化开发任务清单.md`

## 首版边界

首版范围内：

- React 前端
- Flask 后端
- Flask-SocketIO 实时同步
- SQLite 持久化
- Docker + Zeabur 部署

首版不包含：

- 图片地图
- 坐标定位
- 账号体系
- 历史战绩
- 多实例扩容

## 默认引导方式

如果用户只问“这个项目是什么”，先给出一句话定义，然后告诉对方：

- 产品依据优先看 `docs/最终交付确认文档.md`
- 技术实现优先看 `docs/技术方案.md`
- 具体开发从 `docs/开发任务清单.md` 开始
- 如果是前端交付、UI 重构或页面体验问题，优先看 `docs/前端产品化改造设计.md`
