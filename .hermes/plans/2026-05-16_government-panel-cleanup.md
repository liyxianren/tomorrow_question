# 议会大厅 UI 重构：统一行政力语义 + 布局梳理

## 目标

消除议会大厅中「行政力 / 行政点 / 政府财政」三个概念的混乱，统一为**行政力**，并重新组织页面布局让信息流清晰。

## 当前问题

### 1. 概念混乱（三个名词混用）

| 当前用词 | 实际含义 | 问题 |
|---------|---------|------|
| 行政力 | admin capacity（容量上限） | ✅ 可用 |
| 行政点 | 同上，但策略行动中说"1 行政点" | ❌ 与行政力是同一物却叫法不同 |
| 政府财政 | governmentFiscal（花钱的池子） | ✅ 但和行政力关系表述不清 |
| "购买行政力" | 花财政买行政力 | ✅ 但位置在底部，不够突出 |

**用户期望**：所有消耗统一说 **行政力**，财政是获取行政力的手段（购买）。

### 2. 布局问题

当前自上而下顺序：
```
议会大厅 header（政府财政 + 基础 X/Y）
StatStrip（行政力 / 剩余行政力 / 已完成改革 / 现行政策）
思潮信号
改革路径（含三条路径 tab）
策略行动（左侧单独一列，右边空 → 横排撑开 bug）
行政力购买 + 国家能力（右侧列）
现行政策 / 可激活政策
```

问题：
- 「策略行动」用了 `gov-policy-split`（两列布局），但只放了 `gov-policy-split__left`，缺少 `gov-policy-split__right`，导致左侧内容被拉伸成横排
- 「行政力购买」埋在最底部，用户找不到
- 「策略行动」和「政策」语义相近但分开在两个 split 容器中，关联不清

### 3. "市场调节" 残留

`DecisionResourceBar.tsx` 第 40 行仍显示 `市场调节 X/Y` 文本。

## 改造方案

### 新布局（自上而下）

```
┌─ 议会大厅 ──────────────────────────────────┐
│ 政府财政 X  ·  行政力 X （+Y 已购）           │
│ （一句话说明：花政府财政购买行政力，推进改革/策略/政策）│
├──────────────────────────────────────────────┤
│ 💰 购买行政力（stepper）                      │  ← 移到最顶
├──────────────────────────────────────────────┤
│ 🧭 思潮信号                                   │
├──────────────────────────────────────────────┤
│ 🧭 改革路径（三 tab）                          │ 消耗行政力
├──────────────────────────────────────────────┤
│ 🎯 策略行动                                   │ 消耗行政力 + 财政
│ （去掉 split 布局，改为正常单列）              │
├──────────────────────────────────────────────┤
│ ⚙️ 政策                                       │ 消耗行政力/回合
│   现行政策 + 可激活政策                         │
├──────────────────────────────────────────────┤
│ 🎴 国家能力                                   │
└──────────────────────────────────────────────┘
```

### 具体改动

#### A. 术语统一

| 旧 | 新 |
|----|-----|
| "行政点" | "行政力" |
| "行政力不足" / "行政点数不足" | 统一为 "行政力不足" |
| "每项消耗 1 行政点 + 财政" | "消耗 1 行政力 + 政府财政" |
| "消耗 1 行政点" | "消耗 1 行政力" |

#### B. 头部重组

```tsx
// 当前头部
政府财政 X
基础 X/Y

// 改为
政府财政 {fiscal}  ·  行政力 {capacity}（+{purchased} 已购）
<p class="hint">用政府财政购买行政力，推进改革、策略和政策</p>
```

移除单独的 StatStrip（行政力数值已在头部），改为紧凑一行显示思潮警戒线。

#### C. 行政力购买移到顶部

将「💰 提升行政力」section 从底部 `gov-policy-split__right` 移到 header 下方第一位。

#### D. 策略行动布局修复

```diff
- <div className="gov-policy-split">
-   <div className="gov-policy-split__left">
      <h4>🎯 策略行动</h4>
      ...
-   </div>
- </div>
+ <div>
+   <h4>🎯 策略行动</h4>
+   <p>消耗 1 行政力 + 政府财政，当回合生效。</p>
+   <div className="government-actions">
+     ...
+   </div>
+ </div>
```

#### E. 政策合并为单一区域

将「现行政策」和「可激活政策」合并到一个区域，不再分左右列。

#### F. 删除 DecisionResourceBar 中的 "市场调节" 显示

`DecisionResourceBar.tsx` line 40：`breakdown` 中去掉「市场调节 X/Y」部分，改为仅显示「基础 X/Y」。
